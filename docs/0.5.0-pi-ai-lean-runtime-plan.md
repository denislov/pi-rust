# `pi-ai` Lean Runtime Plan For 0.5.0

## Status And Scope

Target workspace version: **`0.5.0`**. Status: **Completed on 2026-07-20**.
The workspace and every inherited package are at `0.5.0`; every ledger row is
closed and the release gates pass.

This plan is the first release plan after the completed reduced `0.4.x` train.
The skipped `0.4.3` through `0.4.5` Extension plans remain historical Skip
records and are not resumed or implied by this work.

The plan is subordinate to the normative ownership and dependency contracts in
[`architecture.md`](architecture.md). `pi-ai` continues to own provider-neutral
model/conversation/stream contracts, provider wire mapping, authentication
inputs, scoped registration, and HTTP/SSE transport. Agent sessions, product
events, operation policy, persistence, CLI/RPC, and TUI behavior remain outside
the crate.

## Decision Summary

`0.5.0` removes Bedrock support completely instead of feature-gating it. The
removal includes the Bedrock provider, AWS credential and SigV4 implementation,
AWS dependencies, Bedrock-specific authentication branches, tests, fixtures,
catalog records, and the `bedrock-converse-stream` built-in API registration.

This is an intentional capability removal. It supersedes only the Bedrock
implementation outcome of `0.3.1` `PAIR-008`; it does not weaken the general
credential redaction, provider terminal, cancellation, timeout, retry, or
transport contracts established by that release. A migration note must state
that applications requiring Bedrock need to remain on `0.4.2` or supply a
separately maintained custom `ApiProvider`; `pi-ai` does not retain a dormant
wire DTO or compatibility shim.

The remaining providers stay inside `pi-ai`. Their protocol-specific wire DTOs
and event interpretation remain provider-owned, while repeated provider-neutral
stream accumulation and terminal mechanics converge behind private helpers.

## Objectives

- remove Bedrock/AWS code, dependencies, models, tests, and advertised API as
  one atomic capability deletion;
- reduce provider implementation duplication without erasing protocol-specific
  semantics;
- make private implementation visibility truthful and remove broad dead-code
  allowances;
- delete private image-generation experiments that have no production path;
- reduce test scaffolding duplication without reducing the provider behavior
  matrix;
- review the stable `pi_ai::api` facade and remove only contracts approved by an
  explicit migration decision;
- preserve scoped `AiClient`/`ProviderRegistry`, cancellation, streaming,
  redaction, and offline deterministic tests;
- produce version, changelog, API snapshot, migration, and architecture evidence
  together.

## Non-Goals

- moving model selection, credential precedence, retry policy, or provider
  registration into `pi-agent-core` or `pi-coding-agent`;
- moving provider wire types across a crate boundary;
- replacing typed wire DTOs with a cross-provider untyped JSON event engine;
- removing tests merely to reduce repository line count;
- reviving process-global provider state or exposing transport implementation
  modules;
- resuming skipped Extension work from `0.4.3` through `0.4.5`;
- preserving Bedrock source, model records, authentication branches, or inert
  compatibility aliases for source compatibility.

## Execution Record

### 2026-07-20: Bedrock/AWS and private image experiment removal

- Deleted the complete Bedrock provider tree, its internal behavior/auth/SigV4
  tests, all AWS dependencies, the HTTP 0.2 compatibility alias, and retired
  authentication/stream-option fields.
- Removed `bedrock-converse-stream` from built-in registration and filtered all
  90 `amazon-bedrock` records from the generated catalog. The catalog now has
  831 records; the catalog generator filters both retired identities and owner
  tests reject either identity if it reaches generated output.
- Added a source boundary guard covering the provider directory, manifest,
  built-in registration, authentication, stream options, transport validation,
  and generated catalog.
- Deleted the test-only image-generation DTO/OpenRouter mapper and tests. The
  conversation `ContentBlock::Image` contract remains intact.
- Removed the provider-tree-wide dead-code/unused-import allowance. The 14 wire
  fields intentionally retained for typed deserialization now use field-local
  dead-code annotations; strict `pi-ai` Clippy passes.
- Added private shared start-event, tool-argument assembly, and required
  `[DONE]` helpers used by the retained OpenAI Completions and Mistral streams.
  Protocol-specific parsing and finish-reason mapping remain provider-owned.
- Current post-removal evidence: `cargo tree -p pi-ai --edges normal --prefix
  none` emits 259 nodes/lines; `generated.json` is 451,965 bytes; and `cargo
  test -p pi-ai --all-features` passes 182 unit tests plus all integration and
  boundary tests. No live provider credentials were used.
- Reproducible baseline/current measurements are recorded in
  [`0.5.0-pi-ai-baseline.md`](0.5.0-pi-ai-baseline.md). Production Rust source is
  down 16.4%, provider source 20.3%, normal dependency-tree output 50.7%, clean
  check time 43.5%, and dedicated target bytes 27.7%.
- The stable facade classification is recorded item-for-item in
  [`0.5.0-migration-guide.md`](0.5.0-migration-guide.md). Only the intentional
  Bedrock option/capability removals are accepted public breaks.
- The retained provider matrix passed all 182 `pi-ai` unit tests and all API/
  boundary tests. `pi-agent-core --all-features` passed fully. The first
  `pi-coding-agent --all-features` run passed 948/949 library tests and failed
  one unrelated render-coalescing timing threshold; its focused rerun passed.

### `PAIR-L002`: Bedrock And AWS Removal

- Delete `src/providers/bedrock/` in full: provider, conversion, event-stream
  decoder, authentication, SigV4, and wire types.
- Remove `bedrock-converse-stream` from built-in API registration and provider
  contract matrices.
- Remove `aws-config`, `aws-credential-types`, `aws-sigv4`, the `http` 0.2 alias,
  and `time` when no retained code needs them.
- Remove Bedrock/AWS environment resolution and auth branches, including tests
  and test-only environment guards that become unused.
- Preserve general secret redaction and explicit missing-credential diagnostics
  for retained providers.
- Add boundary tests proving no `bedrock`, `aws_*`, AWS profile/region variable,
  SigV4, or `bedrock-converse-stream` implementation claim remains in `pi-ai`.
- Do not leave feature flags, stub providers, compatibility aliases, or DTO-only
  remnants.

### `PAIR-L003`: Bedrock Model Removal

- Update the model generator/source inputs so regeneration, rather than an ad
  hoc edit of generated JSON, removes every Bedrock model.
- Regenerate `src/model/generated.json` and prove no record uses provider
  `amazon-bedrock` or API `bedrock-converse-stream`.
- Keep catalog ordering, identity, cost, token-limit, compatibility, and
  supported-API validation deterministic.
- Update CLI/model-list fixtures and snapshots affected by the smaller catalog.
- Document the removed models and custom-provider migration option.

### `PAIR-L004`: Truthful Internal Visibility

- Reduce provider `convert`, `stream`, `wire`, DTO, and helper visibility to the
  narrowest owner that consumes them.
- Remove the provider-tree `#[allow(dead_code, unused_imports)]` from `lib.rs`.
- Use a narrow serde-field allowance only where a deserialized wire field is
  intentionally retained and owner-tested.
- Add source guards preventing implementation modules from becoming a second
  public facade.

### `PAIR-L005`: Retained Provider Stream Convergence

- Inventory repeated message initialization, content-block lifecycle,
  tool-argument assembly, usage/cost application, terminal observation, and
  error construction before introducing helpers.
- Add small private provider-neutral helpers only where at least two retained
  providers share identical invariants.
- Keep each provider's wire event parsing, stop-reason mapping, unknown-event
  policy, and required terminal condition in its own module.
- Preserve exactly one terminal event, fail-closed truncation, cancellation,
  timeout, retry-before-exposure, hook, and diagnostic behavior.
- Do not create a generic provider DSL, dynamic JSON dispatcher, public stream
  builder, or trait hierarchy solely to reduce line count.

### `PAIR-L006`: Private Image Experiment Removal

- Delete the test-only image-generation DTOs, OpenRouter image mapper, and their
  owner tests when the inventory confirms there is still no production
  client/provider/registry path.
- Preserve multimodal conversation image content blocks; image generation and
  conversation image input are separate contracts.
- Retain the facade guard proving `pi_ai::api::images` is not advertised.

### `PAIR-L007`: Stable Facade Audit

## Quantitative Release Targets

Final thresholds are frozen by `PAIR-L001`, with these planning targets:

- remove all 1,507 baseline Bedrock provider lines plus related auth/tests;
- remove all direct AWS SDK and HTTP 0.2 compatibility dependencies;
- reduce retained provider stream-mechanics duplication by at least 20 percent
  without reducing behavior-matrix cases;
- reduce production-related `pi-ai` Rust source by at least 15 percent from the
  reproducible baseline;
- remove the broad provider-tree dead-code allowance;
- preserve or improve retained-provider clean-build time and release artifact
  size;
- introduce no new dependency from `pi-ai` to another workspace crate.

## Validation Gates

Iteration uses the smallest relevant test. Release completion requires at least:

```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test -p pi-ai --all-features
cargo test -p pi-agent-core --all-features
cargo test -p pi-coding-agent --all-features
cargo test --workspace --all-features
scripts/architecture-gates.sh
scripts/release-gates.sh
```

The release record must include the baseline comparison, dependency-tree
comparison, API snapshot result, offline provider matrix, downstream boundary
tests, and an explicit statement that no live provider credentials were used.
