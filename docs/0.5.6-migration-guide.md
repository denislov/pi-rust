# Migration Guide: 0.5.5 to 0.5.6

The 0.5.6 release changes fullscreen TUI presentation and generic editor
composition. It does not change agent, delegation, authorization, session,
persistence, ProductEvent, UI snapshot, or RPC semantics.

## Fullscreen Interaction And Layout

Slash-command and file-completion assistance is now attached to the composer
text origin instead of appearing as a centered transient surface. Modal
settings, model, session, and tree surfaces retain their deliberate modal
placement. Extensions or tests that compare terminal captures should update
expected menu coordinates and selection markers.

Context has three explicit responsive presentations:

- wide terminals use a persistent right rail with a visible Context/Tips
  divider;
- medium terminals use a bordered right drawer closed with `Esc`;
- narrow terminals use a full Context page and hide the parent composer while
  that page owns focus.

Resizing preserves the active tab/item, focus restore target, editor state,
transcript selection, and scroll state. Context tabs may use controlled compact
labels at reduced widths rather than overlapping or splitting graphemes.

Focus and selection no longer share a single `>` glyph. The composer keeps `>`;
focused panels and transcript selection use a left bar, while list and
autocomplete selection use `›` plus theme selection styling. Color-disabled
rendering retains these structural markers.

## Theme And Status Expectations

The product uses the existing resolved theme tokens rather than new hard-coded
colors. Custom themes should provide useful contrast for `accent`,
`borderAccent`, `borderMuted`, `muted`, selected-row, success, warning, and
error roles. Color reinforces state but is not its only signal.

The fullscreen status row no longer displays `session_label` or a durable
session ID. Session identity remains available through `/session` and Context.
Runtime state and a bounded context-usage gauge have highest priority, followed
by profile/model information; cwd abbreviates or disappears first. Capture
tests that expected pipe-separated diagnostic text should update to the new
semantic spacing and gauge.

Startup text is shorter, and System, Assistant, Tool, Error, delegation, and
child-page transcript blocks now reserve one common selection gutter. Markdown
tables preserve their allocated padding and borders as atomic prewrapped lines;
very narrow tables use a deterministic fallback instead of a clipped box.

## Generic `pi-tui` Editor API

`pi_tui::Editor` adds two public composition methods:

- `render_input(width)` renders only the editor input rows;
- `render_assistance(width)` renders only the active autocomplete rows.

The existing `Component::render(width)` behavior remains available and combines
both portions, so ordinary editor embedders do not need to change. Product
shells that need to position assistance separately may call the two methods and
place the returned lines in distinct layout rectangles. No product command,
Context, or agent type moved into `pi-tui`.

## Protocol And Schema Inventory

| Protocol family | 0.5.5 | 0.5.6 |
| --- | --- | --- |
| RPC | `2.1` | `2.1` |
| ProductEvent | `2.2` | `2.2` |
| UI snapshot | `2.2` | `2.2` |

All visual state remains client-local. Session/outbox/manifest records and
Extension schemas are unchanged, so no persistence migration or protocol
negotiation change is required.
