# Migration Guide: 0.5.2 to 0.5.3

The 0.5.3 release hardens the fullscreen terminal runtime and provider stream
validation. It does not change the supported embedding facade or serialized
RPC, ProductEvent, or UI snapshot versions.

## Embedding And CLI

No public import migration is required. Continue to use the categorized
`pi_coding_agent::api` facade and the high-level
`api::cli::runner::{run_cli, run_cli_stdio}` entrypoints. The user-facing binary
reports `0.5.3`; the root `pi-rust` binary remains a placeholder.

Interactive embedders should treat the runtime client connection as the sole
snapshot/replay/live/ack projection path. The fullscreen product adapter no
longer maintains an operation-local ProductEvent subscription. This is an
internal ownership convergence and does not add a public adapter API.

`pi_tui::api::terminal::ProcessTerminal` now implements the auto traits
`UnwindSafe` and `RefUnwindSafe` because the independent progress thread state
was removed. This is additive; its methods and `Terminal` contract are
unchanged.

## Provider Stream Validation

OpenAI Completions and Mistral streams now reject nonzero top-level choice
indices and tool-call kinds other than `function`. Anthropic streams reject a
content-block delta or stop whose index does not match the active block. These
payloads were previously accepted and merged into one output, which could
silently corrupt ordering.

Unused response decoration (`role`, message `type`, stop-sequence text,
redacted-thinking payload, and duplicate Responses item content) is ignored by
serde rather than stored. OpenAI completion reasoning-token details are also
ignored because the provider-neutral `Usage` contract has no corresponding
field. Normal payloads and unknown compatible fields remain accepted.

## Protocol Inventory

| Protocol family | Version |
| --- | --- |
| RPC | `2.1` |
| ProductEvent | `2.2` |
| UI snapshot | `2.1` |

No JSONL client migration is required. Terminal rendering, queue bounds,
viewport caching, and resize scheduling are implementation behavior rather than
serialized protocol changes.

## Extension Contract

The ADR-002 grant, lease-backed Host, and minimum Wasmtime runtime are retained.
Contribution productization/dispatch remains explicitly Skipped. No Extension
manifest, lock, WIT, grant, activation, or trusted-host API migration is
introduced by 0.5.3.
