# Agent And Workflow Convergence Plan For 0.4.1

## Status And Scope

Target workspace version: **`0.4.1`**. Status: **planned and dependency-reviewed;
blocked by completed `0.4.0`**.

This release moves execution semantics onto the `0.4.0` ownership framework. It
types the Agent loop, defines active cancellation, replaces fixed product Flows,
closes `pi-agent-core` correctness gaps, and converges compaction. It does not
introduce third-party Wasm execution.

See the [`0.4.x` roadmap](0.4.x-architecture-extension-platform-roadmap.md).

## Objectives And Non-Goals

- define one cancellation taxonomy across provider, hooks, tools, states/steps,
  structured children, and commit boundaries;
- make production Agent transitions typed and exhaustive;
- use typed async pipelines and structured concurrency for fixed product work;
- decide and converge generic Flow ownership;
- replace Agent admission panics, duplicate IDs, malformed terminals, Unicode
  validation defects, and high-value string errors;
- converge runtime/durable compaction without changing `0.4.0` ownership.

Manifest/WIT/Wasm, new extension services, and Workbench remain out of scope.
Frozen Lua APIs receive no new capability before removal in `0.4.2`.

## Detailed Acceptance

Task-ledger row order is the implementation order. Detailed sections remain in
stable numeric ID order for cross-version traceability.

### `AWC-001`: Cancellation Contract

- Classify provider streaming, hooks, sequential/parallel tools, queued input,
  typed states, workflow steps, structured children, and commit as cooperative or
  host-enforced.
- Every required bound has a deadline/containment mechanism and measured latency.
- Cancellation resolves Aborted; committed effects are not rolled back by fiction.
- Tests cancel active work, not only pre-cancelled tokens.

### `AWC-002`: Typed Agent State Machine

- Replace production string node/action dispatch with exhaustive state/transition
  enums.
- Preserve streaming, queues, thinking, tool ordering, hooks, max turns,
  compaction, and event order.
- Every provider terminal maps to a typed transition or protocol error.
- Transition loops have bounded guards and state-aware diagnostics.
- Private state IDs do not become product API.

### `AWC-003`: Typed Product Workflows

- Export, manual compaction, and other fixed sequences become typed async code.
- Team/member parallelism uses bounded structured children.
- One typed step observer preserves progress, phase, timing, cancellation, and
  errors without graph construction.
- Lifecycle/outbox behavior remains owned by the `0.4.0` framework.

### `AWC-004`: Generic Flow

- Inventory and classify every execution form as dynamic state machine, fixed
  pipeline, structured concurrency, genuinely durable workflow, or test-only
  graph before production migrations begin.
- Record one ADR: stable graph runner, internal/test support, explicit rename, or
  removal.
- If retained, Flow is an in-memory cooperative graph runner, not a durable engine.
- Remove string actions from stable use or replace them with typed transitions.
- Specify missing transition, max-step, callback, cancellation, and errors.
- No product Flow remains merely to wrap sequential calls.

### `AWC-005`: Core Correctness

- concurrent `prompt`/`run` returns typed busy/admission errors;
- all prompt/queue/assistant/tool/compaction/replay message IDs remain unique;
- `ToolUse` without valid calls fails deterministically;
- resource metadata Unicode/length validation is correct;
- stable-boundary Agent errors are typed where valuable;
- one cancellation matrix covers all retained execution forms.

These defects close before the typed Agent migration so the new state machine
does not preserve duplicate identity, panic, malformed-terminal, or validation
behavior as a new contract. Post-migration tests then prove the fixes remain
closed.

### `AWC-006`: Compaction

- Distinguish ephemeral runtime-context and durable session-history compaction.
- Decide automatic compaction durability in persistent sessions.
- Durable compaction uses SessionCoordinator/outbox before success; ephemeral
  compaction emits no durable-looking completion.
- Rehydrate committed summaries across prompts/restart.
- Preserve tool-call context and usage anchors.
- Prevent unintended repeated summarization in consecutive-prompt/restart tests.

### `AWC-007`: Evidence

- Run typed transition, active cancellation, workflow, lifecycle, outbox,
  compaction, and adapter matrices.
- Compare admission/turn/workflow/cancellation baselines with `0.4.0`.
- Set provisional no-major-regression guards for admission, turn execution,
  active cancellation latency, and workflow overhead.
- Update architecture evidence and API snapshots.
- Publish no extension-facing execution seam yet.

### `REL-041-001`: Release Convergence

- Close all `AWC-D*` debt.
- Update all workspace packages to `0.4.1` after implementation/debt closure.
- Add root and owning-crate `0.4.1` changelog sections.
- Record Flow/Agent/compaction API and protocol decisions/migrations.
- Run focused and full release gates.

## Validation And Release Gates

Focused tests cover `pi-agent-core`, operation/session/event snapshots, active
provider/hook/tool cancellation, workflows, compaction, RPC, and public APIs.

```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace --all-features
scripts/tui-smoke.sh
```

`0.4.1` completes only when all tasks/debt close, `0.4.0` lifecycle/outbox tests
still pass, no fixed workflow uses dynamic Flow without ADR, versions/changelogs/
protocols are current, provisional performance guards pass, and ordinary tests
remain offline.
