# `pi-coding-agent` Lean Product Runtime Plan For 0.5.2

## Status And Scope

Target workspace version: **`0.5.2`**. Status: **Completed 2026-07-20;
`PCLR-001` through `PCLR-011` are complete**.
This plan begins only after the `0.5.0`
`pi-ai` and `0.5.1` `pi-agent-core` lean-runtime plans are complete, their
migrations are applied, and their API baselines are frozen. The `0.5.0`
and `0.5.1` dependencies are complete; the workspace is versioned `0.5.2`, and
release evidence is recorded in
[`0.5.2-release-evidence.md`](0.5.2-release-evidence.md).

This plan reduces product-runtime duplication while preserving the normative
owner graph in [`architecture/runtime.md`](architecture/runtime.md):

```text
Adapter
  -> OperationSupervisor
  -> Typed Operation
  -> SessionCoordinator
  -> committed outbox
  -> EventHub
  -> ClientProjectionCoordinator
  -> TUI / RPC / JSON / Embedding
```

`SessionEvent` remains durable session truth. ProductEvents and snapshots remain
the adapter boundary. `RuntimeHost`, operation admission/finalization, the
bounded session writer, outbox/recovery, client projection, profiles,
authorization, extensions, CLI/RPC, and TUI product semantics stay in
`pi-coding-agent`; provider-neutral Agent behavior and provider wire behavior do
not move upward or downward to save adapters.

## Extension Decision

**The TypeScript/Wasm Extension framework is retained.** `0.5.2` does not delete,
stub, default-disable, or misadvertise Extension installation, grants,
activation, invocation, or PluginLoad lifecycle. The accepted Extension
architecture and security boundaries remain normative:

- TypeScript is the authoring language and Wasm Component is the installed
  executable artifact;
- Manifest/WIT/schema contracts remain independently versioned;
- package quarantine, digest/lock verification, immutable installation, and
  activation remain enforced;
- requested permissions, persisted grants, instance generations, and
  operation-scoped leases remain distinct;
- every Host call validates operation, generation, scope, permission, deadline,
  cancellation, and late-result fencing;
- each invocation receives an isolated Store/Instance with fuel, epoch, memory,
  output, and deadline bounds;
- ambient WASI, native/dynamic loading, raw Rust service access, detached guest
  tasks, and unbounded channels remain prohibited;
- PluginLoad durable facts, outbox/terminal association, restart activation,
  diagnostics, and public Extension contracts remain supported.

Extension code may participate in ordinary visibility, projection, test-fixture,
and duplication cleanup, but release acceptance cannot reduce its isolation,
authorization, redaction, package integrity, cancellation, or durability
coverage. Wasmtime and the Rust 1.94 minimum remain intentional product costs.

The contribution dispatch, advanced services/state, Workbench, and complete
Extension DX work skipped after `0.4.2` remain Skipped. Retaining the minimum
framework does not silently resume or claim those features.

## Objectives

- remove the zero-state `WorkflowService` pass-through layer;
- express fixed typed operations as direct async pipelines without recreating a
  generic Flow or string-action engine;
- reduce duplicate operation/outcome/adapter projections while preserving the
  descriptor table and terminal owner;
- contract `pi_coding_agent::api` to supported product verbs, outcomes, events,
  snapshots, protocol negotiation, high-level CLI entrypoints, Extension
  contracts, and narrow fixtures;
- make TUI/RPC/JSON consume common public ProductEvent/snapshot/submission
  semantics rather than internal event or runner paths;
- separate TUI-local state from shared runtime projection and replace mutually
  exclusive pending fields with one typed command;
- converge product tool output shaping on the retained `pi-agent-core`
  execution contract established by `0.5.1`;
- remove debug-build test support and crate-wide Clippy suppressions;
- consolidate repeated fixtures and boundary scanners without reducing durable,
  recovery, protocol, TUI, or Extension coverage;
- preserve the Extension framework and its complete current security contract;
- complete version, changelog, migration, API snapshot, architecture, and
  release evidence together.

## Non-Goals

- deleting, stubbing, default-disabling, or moving the Extension/Wasm runtime;
- weakening package quarantine, grants/leases, Host-call checks, Wasmtime
  containment, PluginLoad durability, or Extension public contracts;
- implementing the previously Skipped contribution/Workbench/services/state/DX
  roadmap;
- collapsing `RuntimeHost`, `OperationSupervisor`, `SessionCoordinator`,
  `EventHub`, `OutboxPublisher`, or `ClientProjectionCoordinator` merely to
  reduce struct count;
- merging SessionEvents, ProductEvents, outbox records, snapshots, or protocol
  DTOs into one unversioned representation;
- moving product policy or adapter state into `pi-ai`, `pi-agent-core`, or
  `pi-tui`;
- adding a generic workflow trait, graph, service container, reflection-based
  operation dispatcher, or string action compatibility layer;
- deleting tests solely to reduce repository line count;
- changing machine-readable stdout/stderr or JSONL cleanliness.

## Detailed Acceptance

### `PCLR-001`: Compiler-Aware Baseline And Inventory

- Measure production, unit-test, integration-test, feature-gated support, and
  generated code separately; do not infer production LOC from the first
  `#[cfg(test)]` occurrence.
- Inventory every public `pi_coding_agent::api` symbol and all workspace
  consumers, including the binary, examples, fixtures, and protocol snapshots.
- Record direct/transitive dependency trees, clean debug/release build time,
  artifact size, peak runtime allocations, snapshot/event projection cost, and
  TUI idle/prompt rendering cost under
  `target/perf-baseline/0.5.2-coding-agent/`.
- Record operation-kind projection sites, runner step/context fields, adapter
  outcome matches, public/internal event conversions, test fixture families,
  and crate-wide lint suppressions.
- Record Extension cold/warm component preparation and invocation baselines
  separately so ordinary cleanup cannot hide a security/performance regression.

### `PCLR-002`: Remove `WorkflowService`

- Delete the stateless `WorkflowService`, its `RuntimeHost` field, constructor
  wiring, and pass-through methods.
- Let each operation module invoke its dedicated runner and normalize its own
  outcome/error exactly once.
- Keep session writes, outbox staging, event publication, authorization,
  capabilities, provider runtime composition, and Extension platform access
  behind their existing owning services/coordinators.
- Preserve cancellation propagation and the distinction between root,
  sub-operation, and structured child invocation.
- Do not replace the service with a generic runner registry, trait object,
  function map, or mutable service container.

### `PCLR-003`: Direct Typed Fixed Pipelines

- Inventory runner state machines and classify each as fixed sequence, dynamic
  loop, or structured concurrency before editing.
- Convert fixed Export, ManualCompaction, BranchSummary, SelfHealingEdit, and
  eligible AgentInvocation/AgentTeam setup/finalization sequences from
  `Step enum + loop + unreachable` into direct typed async calls.
- Keep PromptTurn's dynamic provider/tool/control loop specialized. Keep team
  member execution bounded and structured.
- Remove context `Option` fields and `failure_error` slots that exist only to
  support artificial step re-entry; retain fields required across real async,
  transaction, cancellation, or finalization boundaries.
- Check cancellation at the same or stronger boundaries and preserve exact
  outcome, ProductEvent, SessionEvent, outbox, and finalization order.
- Do not add a shared generic pipeline executor or observer abstraction.

### `PCLR-004`: Operation And Terminal Projection Convergence

- Keep the exhaustive operation descriptor table as the sole metadata authority
  for kind, outcome family, terminal policy, session access, admission class,
  dispatch, durability, cancellation, and child policy.
- Remove duplicate/dead accessors and hand-maintained projections that can be
  derived from the descriptor or typed operation payload.
- Define each public-to-internal operation conversion and internal-to-public
  outcome conversion once; adapters must not repeat operation-family
  `unreachable!()` assumptions.
- Preserve immutable `OperationExecution`, operation/root/parent identity,
  capability generation, idempotency, and finalization evidence.
- Preserve the single terminal authority: runners produce typed outcomes,
  `OperationSupervisor` freezes finalization, and SessionCoordinator validates
  durable evidence without reclassifying it.
- Keep durable/versioned ProductEvent terminal and outcome acknowledgement
  semantics distinct.

### `PCLR-005`: Stable Facade Contraction

- Classify every public symbol as product verb, typed outcome/event/snapshot,
  protocol negotiation/wire contract, retained Extension contract, high-level
  CLI runner, narrow fixture, or migration candidate.
- Preserve `runtime`, `operation`, `event`, `client`, required `view`, versioned
  `protocol`, Extension installation/grant/activation, and high-level CLI runner
  surfaces.
- Remove low-level CLI parsing internals, config-path machinery, settings merge
  implementation, theme parser/token/watcher internals, resource loader
  internals, builtin tool construction, and adapter implementation details from
  the stable facade unless external support is explicitly justified.
- Delete the empty `api::testing` module until a real narrow fixture contract is
  introduced.
- Keep implementation modules private and do not restore root compatibility
  re-exports.
- Publish a `0.5.2` migration guide and reviewed API snapshot diff for every
  breaking removal.

### `PCLR-006`: Adapter Submission And Projection Convergence

- Introduce one adapter-layer exhaustive reduction from
  `CodingAgentOperationOutcome` to a typed submitted-operation result; TUI, RPC,
  JSON, and print then project that result without operation-specific
  impossible-variant matches.
- Make all adapters consume the supported public ProductEvent/snapshot/control
  contracts. Delete parallel internal/public adapter entrypoints when the
  underlying event is already the same typed product contract.
- Preserve adapter-specific presentation and machine wire DTOs; do not merge
  versioned RPC/JSON protocol types into ProductEvents.
- Preserve bounded queues, sequence/cursor gaps, detach, reconnect, fresh
  snapshot recovery, control receipts, and stdout/stderr cleanliness.
- Keep adapters out of workflow, repository, provider, and Extension-host
  internals.

### `PCLR-007`: Interactive Client-State Convergence

- Separate editor/focus/scroll/selection/modal/render-cache state into an
  `InteractiveLocalState` owned only by the TUI client.
- Represent shared session/model/capability/operation/usage/file-change state as
  a projection sourced from `CodingAgentSnapshot` plus ordered ProductEvents;
  do not maintain a second session/runtime truth in `InteractiveRoot`.
- Replace mutually exclusive prompt/compact/branch/fork/agent/team/edit/profile
  pending fields with one exhaustive `PendingInteractiveCommand` enum while
  keeping authorization and delegation queues independently typed where they
  may coexist.
- Centralize operation submission and outcome application through `PCLR-006`.
- Keep all product semantics in `pi-coding-agent`; only generic components,
  input, layout, overlays, and terminal lifecycle remain in `pi-tui`.
- Preserve Unicode width, resize, cursor, scrollback, IME/input, mouse hit,
  modal focus, terminal cleanup, and offline tmux smoke behavior.

### `PCLR-008`: Product Tool Output Convergence

- Compare product `tools/output.rs` and per-tool truncation behavior with the
  retained `pi-agent-core::api::execution` contract after `0.5.1`.
- Reuse core format/head/tail/line/capture mechanics only where byte/line,
  Unicode, ordering, and truncation-marker semantics match.
- Keep product-specific tool limits, authorization, filesystem capability,
  display hints, full-output paths, mutation queues, and error projection in
  `pi-coding-agent`.
- Remove duplicate product implementations and tests only after cross-checking
  read/find/grep/ls/shell output snapshots.
- Do not move coding-agent authorization or workspace policy into the core.

### `PCLR-009`: Lints And Test Infrastructure

- Remove `debug_assertions` from the test-support cfg so environment mutation
  and provider guards are absent from ordinary debug product builds.
- Remove the unused `test-harness` feature or fold justified fixtures into the
  explicit non-default `test-support` contract.
- Remove crate-wide allowances for `result_large_err`, `large_enum_variant`,
  `too_many_arguments`, and `collapsible_if`; address owners or document only
  narrow local exceptions.
- Consolidate repeated provider/session/runtime builders, operation case tables,
  event assertions, and boundary source scanners.
- Let API snapshots cover importability inventories instead of repeating large
  hand-maintained symbol lists, while retaining semantic API behavior tests.
- Keep durable commit/recovery/outbox/cursor, operation association, protocol,
  authorization, TUI lifecycle, tool, and Extension behavior matrices.
- Moving tests between files is not counted as source reduction; only removed
  duplication and compile/runtime improvements count.

### `PCLR-010`: Retained Extension Contract Gate

Completion evidence (2026-07-20): all 33 Extension contract tests passed,
including strict manifest/lock validation, package quarantine, immutable store,
activation, grants, generation/revoke/late-result fencing, and bounded Host
handles. The offline TypeScript SDK regenerated and verified the WIT/schema
surface without ambient WASI or native/dynamic loading. The production

prepare/invoke path, within the frozen approximately 187 second cold baseline.
The accepted Wasmtime prototype matrix separately passed isolated fresh Stores,
bounded memory, deterministic fuel exhaustion, epoch interruption, pending Host
deadline cancellation, and disposal before replacement. Public trusted-host
install/activate/reload and durable PluginLoad SessionEvent/outbox/restart tests
also passed. Wasmtime remains pinned to `46.0.1`, Rust 1.94 remains the minimum,
and all previously Skipped Extension productization work remains Skipped.

- Keep every current Extension public type and trusted-host operation reachable
  through the supported facade unless an independently reviewed compatible
  replacement is introduced.
- Preserve PluginLoad admission, RuntimeWrite classification, capability
  generation changes, durable SessionEvent/outbox terminal evidence,
  diagnostics, restart reload, cancellation, and public outcomes/events.
- Re-run Manifest, lock, quarantine, immutable package store, activation,
  permission/grant/lease, revoke/generation, Host handle, WIT import/export,
  component cache, fresh Store/Instance, fuel/epoch/memory/deadline/output, trap,
  cancellation, and late-result matrices.
- Prove no cleanup exposes repositories, provider clients, raw channels,
  RuntimeHost/services, adapter state, mutable operation contexts, native code,
  ambient WASI, or detached guest tasks.
- Preserve Wasmtime `46.0.1` and Rust 1.94 unless a separately reviewed runtime
  upgrade proves contract and fixture parity.
- Record Extension compile/runtime cost transparently; it is an accepted product
  cost and cannot be excluded from release measurements.
- Do not mark any previously Skipped Extension productization task complete.

## Quantitative Release Targets

Final audit correction: PCLR-001 recorded 108,017 raw source lines while also
labeling that value compiler-attributed. The release record preserves both the
raw result and a like-for-like conservative production reconstruction; it does
not claim a percentage derived by mixing them. The frozen absolute cap and all
structural/API/behavioral gates are enforced. See
[`0.5.2-release-evidence.md`](0.5.2-release-evidence.md).

Final thresholds are frozen by `PCLR-001`, with these planning targets:

- reduce compiler-attributed non-test `pi-coding-agent` source by 15 to 22
  percent without deleting retained Extension capability;
- reduce stable public API items by at least 30 percent while preserving product
  verbs/events/snapshots/protocol/Extension contracts;
- remove `WorkflowService`, fixed-runner artificial step state, repeated adapter
  impossible-outcome matches, and ordinary debug-build test support;
- reduce repeated fixture/boundary/API test infrastructure by at least 20
  percent without reducing required behavioral matrices;
- remove crate-wide Clippy allowances and leave only reviewed local exceptions;
- materially reduce InteractiveRoot field count and operation-specific pending
  branches while preserving TUI behavior;
- remove product output-shaping duplication accepted by the `0.5.1` core
  contract;
- preserve all current Extension security tests and keep Extension cold/warm
  runtime and build performance within `PCLR-001` frozen regression bounds;
- introduce no reverse dependency, second durable truth, generic workflow
  engine, or product types in lower crates.

## Validation Gates

Iteration uses the smallest owning test. Release completion requires at least:

```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test -p pi-coding-agent --all-features
cargo test --workspace --all-features
scripts/architecture-gates.sh
scripts/extension-contracts.sh
scripts/extension-sdk.sh run-conformance
scripts/extension-runtime.sh
scripts/release-gates.sh
scripts/tui-smoke.sh
```

The release record must include production/test/API/dependency comparisons,
operation/finalization/session/outbox/recovery/client matrices, adapter protocol
and stdout cleanliness results, TUI smoke captures, and the full retained
Extension security/conformance evidence. Ordinary provider tests remain offline
and require no live credentials.
