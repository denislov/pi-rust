# `pi-agent-core` Lean Runtime Plan For 0.5.1

## Status And Scope

Target workspace version: **`0.5.1`**. Status: **Planned on 2026-07-20;
implementation has not started**. Its `0.5.0` dependency is complete and the
`pi-ai` migration/API baseline is frozen. The released workspace is `0.5.0`;
`0.5.1` is not claimed as current implementation evidence.

This plan reduces `pi-agent-core` to its provider-neutral runtime ownership:
the typed Agent loop, generic tools and execution contracts, hooks, resources,
compaction primitives, and transcript DTOs. It is subordinate to the normative
contracts in [`architecture.md`](architecture.md) and does not move product
sessions, durable facts, operation admission, authorization policy, ProductEvents,
profiles, CLI/RPC, adapters, or TUI behavior into the core.

The plan removes alternative session/branch/test runtimes that are not on the
production path, completes the typed Agent-turn convergence, reduces long-turn
copying, and contracts the stable facade with explicit downstream migration
evidence.

## Objectives

- delete the unused core Branch Summary workflow alternative while retaining
  provider-neutral summarization primitives;
- delete the test-only Session Context/Memory subsystem that resembles a second
  session owner;
- eliminate residual string actions from the typed Agent turn;
- reduce full-history/config/tool/resource cloning and duplicate event storage;
- replace the parallel test Harness/Proxy runtime with tests of the production
  Agent and narrow deterministic fixtures;
- consolidate shared mechanics in skill and prompt-template resource loading;
- reduce duplicate and product-specific facade exports;
- preserve Agent event order, tools, hooks, cancellation, queues, compaction,
  transcript wire contracts, and execution-environment ownership;
- complete version, changelog, migration, API snapshot, architecture, and
  downstream evidence together.

## Non-Goals

- moving the Agent loop, generic tools, execution environment, resources,
  provider-neutral compaction, or transcript DTOs into `pi-coding-agent`;
- moving product session persistence, `SessionEvent`, ProductEvent, admission,
  authorization, profiles, or adapter concerns into `pi-agent-core`;
- introducing a replacement generic Flow/graph/workflow engine;
- retaining test-only runtimes solely because they imitate an earlier
  TypeScript harness;
- deleting behavior tests merely to reduce line count;
- changing provider wire, registry, authentication, or transport ownership;
- resuming skipped `0.4.3` through `0.4.5` Extension work.

## Detailed Acceptance

### `ACLR-001`: Reproducible Baseline

- Recount source by ownership directory after the completed `0.5.0` dependency
  and API changes.
- Inventory every `pi_agent_core::api` item and all production/test consumers in
  `pi-coding-agent`.
- Record clean check/test build time, direct and inherited dependency trees,
  release artifacts, and Agent-turn peak allocations for short and long
  transcripts under `target/perf-baseline/0.5.1-agent-core/`.
- Record message/config/tool/resource clone counts and emitted-event clone counts
  with a deterministic provider/tool fixture.
- Freeze quantitative thresholds from this baseline before behavioral edits.

### `ACLR-002`: Remove The Alternative Branch Summary Workflow

- Delete `compaction/branch.rs`, `branch_error.rs`, their facade exports, and
  tests that only exercise that alternative workflow.
- Remove `BranchSummaryOptions` carrying API key/header inputs, session-tree
  branch selection, file-operation policy, and provider stream lifecycle from
  the core.
- Preserve `serialize_conversation`, token estimation, summarization-context
  construction, and `summarize_with_provider_streamer` as provider-neutral
  primitives used by the Agent and typed product runners.
- Prove `pi-coding-agent` remains the only BranchSummary workflow/session owner
  and does not import a replacement internal core path.
- Do not add compatibility aliases or a second branch runner.

### `ACLR-003`: Remove The Test Session Subsystem

- Delete `context/assembly.rs`, `context/memory.rs`, `context/error.rs`,
  `SessionContext`, `InMemorySessionStorage`, and their `api::testing` exports.
- Preserve `context/conversion.rs`, which owns provider-neutral conversion from
  `AgentMessage` to `pi-ai` conversation values.
- Replace dependent tests with direct `AgentMessage`, transcript, and replay
  fixtures rather than a second in-memory session/storage abstraction.
- Update legacy-session boundary tests to assert the subsystem is absent rather
  than requiring it under a renamed directory.
- Prove no durable/session repository ownership enters the core.

### `ACLR-004`: Exhaustive Typed Agent Turn

- Match `AgentTurnDecision` variants directly; delete all `ACTION_*` strings,
  `as_str()`, `action()`, and unknown-string transition paths.
- Make state/decision combinations exhaustive at compile time and keep the
  bounded state-step guard.
- Introduce a private typed `AgentTurnError` for invariant, provider, hook,
  tool, compaction, and cancellation failures; convert to the stable external
  event/error only at the Agent boundary.
- Keep node/state topology private. Remove node functions, `AgentTurnContext`,
  `PendingToolCall`, runtime-compaction internals, and decisions from the
  `test-support` facade.
- Test transitions through the production Agent stream, with narrowly scoped
  unit tests only for local pure decisions.

### `ACLR-005`: Turn State And Copy Convergence

- Delete the unused `AgentTurnContext.events` mirror; production events flow
  through the Agent stream once and tests collect that public stream.
- Make turn-to-Agent state application consume/move messages and queues instead
  of cloning them back.
- Remove the duplicate `resources` field when `AgentConfig` remains its owner,
  or freeze one immutable resource snapshot without storing both forms.
- Share immutable tools/resources/config components where that reduces copying
  without transferring mutation authority.
- Preserve live steering/follow-up insertion, provider-request override,
  cancellation, concurrent admission rejection, and `Agent::messages()`
  observation semantics.
- Add allocation/copy regressions for multi-turn long transcripts; do not hold a
  blocking `RwLock` guard across an async wait.

### `ACLR-006`: Test-Support Convergence

- Inventory behavioral coverage in `testing/harness.rs` and `testing/proxy.rs`
  before deletion.
- Re-express production-relevant cases with `Agent`, `AgentHooks`,
  `pi_ai::api::testing::FauxProvider`, and small fixture builders.
- Delete the duplicate harness hook/auth/header/phase/subscription vocabulary
  and proxy provider event engine after coverage is transferred.
- Remove the direct `reqwest` dependency from `pi-agent-core` when the proxy is
  gone; networking remains owned by `pi-ai`.
- Retain a narrow `InMemoryExecutionEnv` under non-default `test-support` if it
  continues to provide downstream deterministic value.
- Keep test-support types out of production signatures and ordinary builds.

### `ACLR-007`: Resource Loader Convergence

- Share private deterministic file discovery, UTF-8 read, frontmatter,
  provenance wrapping, and diagnostic collection between skills and prompt
  templates.
- Keep Skill and PromptTemplate naming, validation, collision, recursion, and
  invocation semantics separately typed.
- Do not introduce a loader registry, plugin mechanism, service container, or
  generic resource trait solely to remove two-way duplication.
- Preserve path ordering, first-wins behavior, source tags, Unicode limits,
  diagnostics, and existing project/user/builtin precedence as caller-owned
  policy.

### `ACLR-008`: Facade And Ownership Convergence

- Classify every public item as active product edge, intended embedding API,
  retained test support, or migration candidate.
- Export `AgentResources` canonically from `api::resources`; update duplicate
  `api::agent` consumers through an explicit migration decision.
- Remove deleted branch/session/node/harness/proxy contracts from the facade and
  public API snapshot.
- Move `TreeFilterMode` to `pi-coding-agent`: `/tree` selector filtering is
  product presentation policy, while `SessionEntry`, `StoredAgentMessage`,
  transcript identifiers, and the generic tree DTO remain core-owned.
- Review execution capture/truncation conveniences separately from the retained
  `ExecutionEnv`/`FileSystem`/`Shell` contracts; local non-use is not sufficient
  to delete an intended embedding contract.
- Preserve exact `pi-agent-core -> pi-ai` categorized/item-level allowlists.

### `ACLR-009`: Downstream And Release Convergence

- Update `pi-coding-agent` imports, Tree selector ownership, tests, and boundary
  allowlists without introducing private-module access.
- Add `docs/0.5.1-migration-guide.md` and a reviewed public API diff.
- Refresh `docs/architecture/current-state.md` and architecture ownership tests.
- Generate and verify `docs/api-snapshots/0.5.1/SHA256SUMS`.
- Advance every workspace package to `0.5.1` only after all implementation and
  debt rows close; update root and all crate changelogs truthfully.
- Update release gates while preserving historical `0.4.x` Skip/completion and
  `0.5.0` release evidence.
- Run the full workspace and TUI gates before marking this plan complete.

## Quantitative Release Targets

Final thresholds are frozen by `ACLR-001`, with these planning targets:

- reduce non-`test-support` `pi-agent-core` source by 18 to 25 percent;
- remove approximately 527 lines of alternative Branch Summary code and
  approximately 814 lines of test Session Context code;
- reduce `test-support` implementation by at least 65 percent after coverage
  transfer;
- remove the direct `reqwest` dependency from `pi-agent-core`;
- eliminate string Agent-turn actions and duplicate event retention;
- eliminate the turn-completion clone of the full message/queue state and
  materially reduce long-transcript peak allocation;
- add no reverse workspace dependency or product type to the core;
- preserve or improve the count of production-behavior tests despite reducing
  harness-only tests.

## Validation Gates

Release completion requires at least:

```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test -p pi-agent-core --all-features
cargo test -p pi-coding-agent --all-features
cargo test --workspace --all-features
scripts/architecture-gates.sh
scripts/release-gates.sh
scripts/tui-smoke.sh
```

The release record must include source/dependency/copy/allocation comparisons,
Agent/tool/hook/queue/cancellation/compaction/transcript matrices, API snapshot
and migration results, and explicit confirmation that ordinary tests remained
offline.
