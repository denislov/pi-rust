# Fullscreen Runtime Status Refresh Plan For 0.5.8

## Status And Scope

Target workspace version: **`0.5.8`**. Status: **Completed 2026-07-22**.

This scoped successor to 0.5.7 fixes three related fullscreen presentation
defects: the running status spinner can freeze under continuous client delivery,
running operation elapsed labels update only when another event happens, and
compact Context Operations rows spend scarce width on `op_xxx` identities.

## Root Cause And Decision

The 120ms spinner sleep lived inside `receive_prompt_source()`. The main loop
also selects client ProductEvents; every winning client delivery cancelled and
recreated the nested prompt-source future, resetting its sleep. A busy stream
could therefore starve the tick indefinitely. Operation elapsed values use the
render-time clock, so they froze for the same reason: no periodic render was
requested.

0.5.8 gives the main loop one independent, missed-tick-skipping 120ms interval.
While a PromptTask is running, each tick advances the spinner and requests a
render. Prompt completion/handoff and client delivery remain separate branches
and cannot reset that interval.

Context Operations summaries show status, kind, elapsed, and the available
cancel action. Full operation, parent, and root IDs remain in the selected
detail surface; no protocol or projection field is removed.

## Acceptance

- Streaming prompt/client activity cannot cancel or reset the runtime interval.
- Every running tick advances the status spinner and schedules a render.
- A visible running operation elapsed label changes without user input, resize,
  or a new ProductEvent.
- Tick processing stops requesting periodic work once no PromptTask is running.
- Compact Context Operations rows contain no operation ID, including short
  `op_xxx` forms.
- Operation detail retains full operation, parent, and root identities plus
  status, elapsed, diagnostics, and cancel availability.
- Prompt completion, render coalescing, terminal cleanup, resize, cancellation,
  child isolation, and differential rendering remain intact.
- RPC `2.1`, ProductEvent `2.2`, UI snapshot `2.2`, persistence schemas, and
  public Rust APIs remain unchanged.
- Full release/Extension gates pass and the committed worktree is clean.

## Non-Goals

- Do not change operation identity generation or projection retention.
- Do not add animation when the runtime is idle.
- Do not remove IDs from diagnostic/detail or machine-readable surfaces.
- Do not introduce a second renderer, timer thread, or progress writer.
