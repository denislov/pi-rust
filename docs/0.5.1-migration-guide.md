# Migration Guide: 0.5.0 to 0.5.1

This guide covers the public API changes in the `pi-agent-core` lean-runtime
convergence release.

## Removed: Core Branch Summary Workflow

The unused core Branch Summary workflow alternative has been removed.
`pi-coding-agent` remains the sole BranchSummary workflow/session owner.

### Removed exports from `pi_agent_core::api::compaction`

- `BranchPreparation`, `BranchSummaryOptions`, `BranchSummaryResult`
- `CollectEntriesResult`, `FileOperations`
- `collect_entries_for_branch_summary`, `generate_branch_summary`
- `generate_branch_summary_with_provider_streamer`, `prepare_branch_entries`
- `BranchSummaryError`, `BranchSummaryErrorCode`

### Preserved primitives

- `serialize_conversation`, `build_summarization_context`
- `summarize`, `summarize_with_provider_streamer`
- `estimate_tokens`, `calculate_context_tokens`, `estimate_context_tokens`
- `prepare_compaction`, `should_compact`
- `ContextUsageEstimate`

### Migration

If you imported Branch Summary types from `pi_agent_core::api::compaction`, use
`pi-coding-agent`'s typed BranchSummary runner instead. The provider-neutral
summarization primitives remain available for custom workflows.

## Removed: Test Session Context Subsystem

The test-only Session Context/Memory/Error subsystem has been removed from
`pi_agent_core::api::testing`.

### Removed exports

- `SessionContext`, `InMemorySessionStorage`, `SessionError`, `SessionErrorCode`
- `build_session_context`

### Migration

Use direct `AgentMessage`, transcript, and replay fixtures instead of the
in-memory session/storage abstraction. The `context::conversion` helpers
(`assemble_context`, `convert_to_context`, `default_convert_to_llm`) remain
available under `api::testing`.

## Changed: Agent-Turn Internal API

The typed Agent-turn runner no longer uses string actions. All `ACTION_*`
string constants, `as_str()`, and `action(&str)` have been removed.

### Removed from `api::testing`

- `AgentTurnContext`, `AgentTurnDecision`, `AgentTurnProviderRequestOverride`
- `PendingToolCall`, `RuntimeCompactionState`
- Node functions: `start_turn`, `decide_after_assistant`, `execute_tools`,
  `maybe_compact_runtime_context`, `maybe_prepare_next_turn`,
  `prepare_provider_request`, `stream_provider`, `drain_queued_input`,
  `apply_before_provider_request_hook`

### Migration

Test Agent-turn behavior through the production `Agent` stream with
`pi_ai::api::testing::FauxProvider`. The typed runner's state/decision
topology is now private.

## Removed: Test Harness/Proxy Runtime

The parallel test Harness/Proxy runtime has been removed. The direct `reqwest`
dependency has been removed from `pi-agent-core`.

### Removed from `api::testing`

- `AgentHarness`, `AgentHarnessEvent`, `AgentHarnessHooks`, `AgentHarnessPhase`
- `ProviderAuth`, `ProviderResponse`, `Patch`, `HeaderPatch`
- `BeforeProviderPayload*`, `BeforeProviderRequest*`, `StreamOptionsPatch`
- `AbortResult`, `Observer`, `SubscriptionGuard`, `HarnessContext`
- `on_kind`, `GetApiKeyAndHeadersHook`, `ContextHook`
- `ProxyRequest`, `ProxyMessageState`, `ProxyStreamOptions`
- `ProxyAssistantMessageEvent`, `ProxyTransportFuture`
- `build_proxy_request_body`, `process_proxy_event`
- `stream_proxy`, `stream_proxy_with_transport`

### Preserved

- `InMemoryExecutionEnv` (under non-default `test-support`)
- `assemble_context`, `convert_to_context`, `default_convert_to_llm`
- `bash_execution_to_text`

### Migration

Re-express test coverage using `Agent`, `AgentHooks`, and
`pi_ai::api::testing::FauxProvider`. Provider auth, headers, and stream
options are configured through `AgentConfig` and `StreamOptions`.

## Removed: Public `TreeFilterMode`

`TreeFilterMode` has been removed from `pi_agent_core::api::transcript` because
`/tree` selector filtering is product presentation policy. `pi-coding-agent`
retains a private implementation for its interactive adapter; it is not a
public embedding API.

### Migration

External consumers should own their presentation-specific filter enum and map
it to `SessionTreeNode` filtering behavior. There is no public replacement type
in `pi-coding-agent`. Code inside `pi-coding-agent` uses its private
`adapters::interactive::tree_selector::TreeFilterMode` implementation.

## Added: `BeforeProviderRequestHook` in `api::agent`

`BeforeProviderRequestHook` is now exported from `pi_agent_core::api::agent`
(previously only available through the removed harness facade). This is the
canonical location for the hook type used by `AgentHooks`.
