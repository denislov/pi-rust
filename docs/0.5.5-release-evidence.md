# 0.5.5 Release Evidence

Release date: 2026-07-22. Workspace package and binary version: `0.5.5`.
Implementation baseline: commit `d32367d` plus the scoped 0.5.5 candidate diff.

## Operation-Tree Failure Model

The frozen model is
[`architecture/operation-tree-failure-model-0.5.5.md`](architecture/operation-tree-failure-model-0.5.5.md).
Every admitted tree converges to exactly one of `Committed`, `DefiniteFailure`,
or `RecoveryPending`. It assigns one owner to admission/cancellation,
authorization waiters, child task/guard lifetime, terminal decisions,
persistence/outbox obligations, ProductEvent publication, and adapter
projection.

The implementation and tests prove:

- dropped authorization futures clean their registry entry through RAII;
  delegation-tool and real child-Prompt authorization abort/shutdown paths
  resolve once, reject late decisions, and execute no unauthorized tool;
- final shutdown cancels all open roots and children once, while Phase A closes
  admission, resolves permission waits, and continues to drain ordinary
  admitted provider work;
- stalled provider streams are dropped on parent cancellation; partial output
  followed by error is visible but uncommitted, truncated tool arguments do not
  execute, and the first terminal prevents duplicate/late stream mutation;
- child partial/failure output remains child-scoped and gives the parent one
  typed delegation result without an extra continuation after cancellation;
- pre-persistence append rejection is `SessionWriteRejected`, while manifest
  uncertainty preserves the original `PartialCommit`; authoritative reopen can
  prove committed from terminal facts and durable outbox identity without
  contradictory recovery residue;
- bounded writer saturation, independent progress, close/join, and shutdown
  after command admission all converge; durable outbox redelivery remains
  ordered, idempotent, and gap-aware;
- child projections stay at 32 conversations, 2,048 queued events, and 1,024
  transcript items. The active page is pinned during eviction; root transcript
  and parent model context remain isolated;
- child rendering is bounded at zero/narrow widths with CJK, emoji, combining
  marks, large tool output, and Kitty images. Generic `pi-tui` ownership covers
  fail-on-Nth full/differential/overlay/resize redraw, image-delete retry,
  lifecycle cleanup, and panic unwind.

Test-only immutable resource snapshots report authorization waiter/grant state
and operation root/child/cancel/release state. Boundary guards prove these
observations and store fault controls do not create a second production owner.
