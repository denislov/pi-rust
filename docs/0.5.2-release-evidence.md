# 0.5.2 Release Evidence

Release date: 2026-07-20. Workspace package version: `0.5.2`.

## Quantitative Inventory

| Measure | 0.5.1 planning baseline | 0.5.2 result |
| --- | ---: | ---: |
| Raw `pi-coding-agent/src` Rust lines, excluding `internal_tests` | 108,017 | 102,054 |
| Conservative production upper bound, excluding whole `cfg(test)` modules | 77,073 reconstructed | 76,769 |
| Direct stable-facade exports | 292 | 189 |
| Generated `pi-coding-agent` public-api items | 15,533 | 12,143 |
| Direct production dependencies | 24 | 24 |
| External integration-test source | 22,636 | 22,375 |
| Explicit `src/internal_tests` source | 5,700 | 5,693 |
| `runtime/facade/tests.rs` | 6,203 | 6,196 |

The PCLR-001 record incorrectly described the 108,017 raw-line inventory as
compiler-attributed while also stating that raw counts could not be release
evidence. A like-for-like reconstruction is therefore included above and no
15% source-reduction percentage is claimed. The frozen absolute production cap
of 91,814 lines is satisfied by the conservative 76,769-line upper bound. The
release decision is based on the completed structural removals and executable
boundary gates, not on mixing the raw and compiler-selected denominators.

The facade threshold is met directly: 292 to 189 exports is a 35.3% reduction.
Rustdoc item inventory falls by 3,390 items (21.8%). The lower-layer public API
JSON files remain checksum-identical to 0.5.1. The direct dependency set is
unchanged; the accepted Wasmtime subtree is retained rather than hidden from
the release build.

## Runtime Convergence

- `WorkflowService` and its `RuntimeHost` field are absent; fixed operations
  invoke typed runners directly.
- Fixed step loops, artificial failure take-back state, and adapter
  impossible-outcome matches are removed.
- ProductEvent DTO mapping and operation descriptors have one production owner.
- Interactive pending commands, local adapter state, and ordered shared
  projection are explicitly separated.
- Product tools reuse the `pi-agent-core` output shaping contract.
- `test-harness`, debug-build test support, and crate-wide Clippy allowances are
  absent; only reviewed item-local exceptions remain.
- The stable CLI facade contains only its high-level runner category; low-level
  parser/config/input/resource/theme behavior remains covered by owner tests.

## Extension Evidence

- 33 manifest/lock/package/store/activation/grant/lease/Host contract tests;
- TypeScript SDK and generated WIT/schema conformance;
- production TypeScript-to-Wasm vertical slice, with a 182.23 second debug cold
  prepare/invoke path against the approximately 187 second frozen baseline;
- four Wasmtime prototype cases covering fresh Store memory isolation, memory
  bounds, deterministic fuel exhaustion, epoch interruption, Host deadline
  cancellation, and disposal before replacement;
- public trusted-host install/activate/reload plus durable PluginLoad
  SessionEvent/outbox/restart evidence.

Wasmtime remains pinned to `46.0.1`, Rust 1.94 remains the minimum, ambient WASI
and native/dynamic loading remain prohibited, and skipped Extension
productization work remains skipped.
