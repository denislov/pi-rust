# `pi-rust` 0.1 → 0.2 Public API Diff

> Status: release-candidate input for the breaking architecture convergence project.
>
> Comparison baseline: `605bb8a chore: checkpoint crates before breaking convergence`.
>
> Target: Rust crates `0.2.0`, RPC/ProductEvent/Snapshot protocol family major `2`.

## Scope and evidence

This document records externally relevant differences between the 0.1 baseline and the current convergence branch. Evidence comes from:

- the public exports at `605bb8a` and at the current `pi_ai::api`, `pi_agent_core::api`, and `pi_coding_agent::api` facades;
- `api_boundary_guards`, `public_api`, `product_runtime_boundary_guards`, `protocol_events`, and RPC wire tests;
- the canonical public event, snapshot, reconnect, and RPC command types.

`cargo-public-api` was not available in the workspace, so this is a source-and-contract diff rather than generated rustdoc JSON output. Before the 0.2 release candidate, CI should produce and archive a machine-generated public API snapshot as an additional release artifact.

## Summary

| Area | 0.1 baseline | 0.2 target | Breaking impact |
|---|---|---|---|
| Stable imports | Stable facades existed, but several implementation modules and `pi-coding-agent` root compatibility exports remained reachable | Stable imports go through each crate's `api` module | Old root/module paths stop compiling |
| Provider registry | `pi_ai::registry::*` was public and doc-hidden | `registry` is private; scoped types remain in `pi_ai::api` | Import-path break; global-style access is unavailable |
| Agent-core modules | Flow, transcript, compaction, resources, proxy, session context, and related modules were public/doc-hidden | Modules are private and their supported contracts are re-exported by `pi_agent_core::api` | Import-path break; supported symbols remain available through the facade |
| Coding-agent root surface | Deprecated root exports and doc-hidden utility modules mirrored CLI/runtime/tool symbols | Root exports are deleted; nine utility modules are private and the remaining internal-module test consumers are being converged | Import-path break |
| Product event metadata | Public event carried deprecated string `family` and `kind` fields in addition to typed payload | Only typed event payload/family/kind accessors remain | Struct field and serialized shape break |
| Capability revocation | Public enum included `CancelMatchingOperations` although no runtime cancellation implementation existed | Only `FutureOnly` remains | Enum variant removal |
| Snapshot cursor | Cursor carried sequence and capability generation | Cursor also requires stream identity and snapshot protocol major; wire uses camelCase | Struct construction and wire break |
| Reconnect | Callers could reconnect with a bare event sequence | Canonical reconnect validates a complete cursor | Method/request migration required |
| Submitted operation state | Included unreachable `Accepted` state | Starts at `Running`, then becomes terminal | Enum variant removal |
| RPC prompt replay | `afterSnapshotSequence` accepted a scalar sequence | `afterSnapshotCursor` requires the full cursor object | RPC request break |
| Plugin Flow extension | Internal model advertised arbitrary Flow extension points with no runtime consumer | Model, capability counter, registry storage, and collector are deleted | Internal non-contract surface intentionally unavailable |

## `pi-ai`

### Removed module path

```rust
// 0.1: no longer supported
use pi_ai::registry::{AiClient, ApiProvider, ProviderRegistry};

// 0.2
use pi_ai::api::{AiClient, ApiProvider, ProviderRegistry};
```

The supported facade symbols remain available. The breaking change is that `registry` is now a private implementation module. Provider registration and lookup must be scoped to an `AiClient`/`ProviderRegistry`; consumers cannot rely on module visibility or a global compatibility path.

### Stable facade additions/removals

No intentional symbol removal was made from `pi_ai::api` relative to the baseline. Provider registry implementation visibility is the relevant break.

## `pi-agent-core`

### Modules made private

The following 0.1 doc-hidden module paths are no longer public:

- `pi_agent_core::agent_turn_flow`
- `pi_agent_core::branch_summary`
- `pi_agent_core::compaction`
- `pi_agent_core::flow`
- `pi_agent_core::proxy`
- `pi_agent_core::resources`
- `pi_agent_core::session_context`
- `pi_agent_core::shell_output`
- `pi_agent_core::transcript`
- `pi_agent_core::truncate`
- `pi_agent_core::types`

Supported contracts from those owners are re-exported through `pi_agent_core::api`. Examples include:

- Flow: `Action`, `Flow`, `FlowError`, `FlowNode`, `FlowOutcome`, `FlowRunOptions`, `NodeId`;
- agent turn: `AgentTurnContext`, `AgentTurnFlow`, node contracts, and focused node functions;
- compaction and branch summary contracts;
- proxy streaming contracts;
- resource loaders;
- session context and in-memory storage;
- transcript/session tree types;
- truncation and shell-capture helpers.

### Newly supported facade symbols

The 0.2 facade explicitly adds previously module-scoped supported contracts, including:

- branch-summary types/functions and `BranchSummaryError`;
- compaction estimate/prepare/summarize functions;
- Flow primitives;
- proxy request/stream types and functions;
- session-context types/functions;
- transcript/session metadata and conversion helpers;
- `execute_shell_with_capture`.

These are facade additions, not a promise that their old module paths remain valid.

## `pi-coding-agent` Rust API

### Root modules made private

The following 0.1 doc-hidden paths are no longer public; supported symbols remain available from `pi_coding_agent::api`:

- `pi_coding_agent::args`
- `pi_coding_agent::config`
- `pi_coding_agent::error`
- `pi_coding_agent::input`
- `pi_coding_agent::interactive`
- `pi_coding_agent::models`
- `pi_coding_agent::print_mode`
- `pi_coding_agent::prompt_options`
- `pi_coding_agent::protocol`
- `pi_coding_agent::request`
- `pi_coding_agent::resources`
- `pi_coding_agent::runtime`
- `pi_coding_agent::session`
- `pi_coding_agent::theme`
- `pi_coding_agent::tools`

Closing these modules also exposed and removed two functions with no repository consumer: `resources::print_diagnostics` and `session::session_root_from_run_options`. They are removals, not facade migrations.

The curated facade now explicitly carries the supported configuration/settings types, prompt-input processing contracts, protocol adapter/RPC/JSONL/version contracts, and the complete theme parsing/detection/reload/highlighting slice. This preserves usable public signatures while preventing consumers from reaching unrelated module internals.

`interactive` and built-in tool implementation tests now compile as owner-crate internal tests. Supported tool construction/filtering remains in `pi_coding_agent::api`; individual bash/edit/read/write execution internals and the interactive scripted harness are intentionally not embedding contracts.

### Removed deprecated root exports

The following root imports are deleted. Import them from `pi_coding_agent::api`:

- `CliArgs`, `CliMode`, `help_text`, `parse_args`;
- `CliError`;
- `PrintModeOptions`, `run_print_mode`;
- `PromptRunOptions`;
- `CliRunOptions`, `PromptInvocation`, `SessionMode`, `SessionRunOptions`;
- `DEFAULT_MODEL_ID`, `DEFAULT_SYSTEM_PROMPT`;
- `build_agent_config`, `effective_no_context_files`, `effective_session_dir`, `select_model`;
- `ResolvedSessionTarget`, `encode_cwd`;
- `builtin_tools`.

The root CLI functions and output type are also consumed through the stable facade:

```rust
use pi_coding_agent::api::{
    CliOutput, run_cli, run_cli_with_options, run_cli_with_options_and_stdin,
};
```

### Product event shape

Removed public fields from `CodingAgentProductEvent`:

```rust
pub family: String;
pub kind: String;
```

Use:

```rust
event.event();        // &CodingAgentProductEventKind
event.family_typed(); // CodingAgentProductEventFamily
event.kind_name();    // stable snake_case variant name for display/metrics
```

Consumers should pattern-match the typed family payload enums. Reconstructing control flow from formatted legacy strings is unsupported.

### Capability revocation

Removed:

```rust
CodingAgentProductEventCapabilityRevocation::CancelMatchingOperations
```

The only current public policy is `FutureOnly`: an active operation keeps its frozen capability generation. A future force-revocation feature must add real cancellation/failure semantics before adding a public enum variant.

### Snapshot cursor

0.1:

```rust
CodingAgentSnapshotCursor {
    last_event_sequence,
    capability_generation,
}
```

0.2 target:

```rust
CodingAgentSnapshotCursor {
    stream_id,
    snapshot_protocol_major,
    last_event_sequence,
    capability_generation,
}
```

The serialized field names are:

```json
{
  "streamId": "session-id",
  "snapshotProtocolMajor": 2,
  "lastEventSequence": 42,
  "capabilityGeneration": 7
}
```

The numeric major in this example is the 0.2 release target. Until M7 changes constants, development builds may still report protocol major `1`; callers must use the value received in the snapshot rather than hard-code it.

### Reconnect

Added canonical method:

```rust
connection.reconnect_from_cursor(&cursor)?;
```

It rejects:

- a cursor from a different session stream;
- an incompatible snapshot protocol major.

The older sequence-only reconnect path is not the canonical adapter contract.

### Submitted operation status

Removed unreachable variant:

```rust
CodingAgentSubmittedOperationStatus::Accepted
```

Observable state now starts at `Running` after atomic admission/commit and transitions to terminal evidence or typed uncertainty.

### Removed error surface

`CodingSessionError::StaleClientConnection` was removed with the parallel stale-client registry path. Client lifecycle failures now use the canonical lifecycle/generation rejection contract.

### Operation admission

`CodingAgentOperation` remains the public operation enum. Internally, every variant now has one descriptor containing its admission class, dispatch mode, submitted kind, outcome family, and permitted terminal evidence. Internal scheduler, service, Flow-node, and plugin-registry types are intentionally not exported from the stable facade.

## RPC and protocol wire changes

### Prompt reconnect field

Removed:

```json
{"afterSnapshotSequence": 42}
```

Required replacement:

```json
{
  "afterSnapshotCursor": {
    "streamId": "session-id",
    "snapshotProtocolMajor": 2,
    "lastEventSequence": 42,
    "capabilityGeneration": 7
  }
}
```

### Product events

Consumers must decode the typed event payload. The removed top-level legacy `family`/`kind` string fields are not emitted as compatibility duplicates.

### Protocol major release boundary

The convergence plan targets RPC, ProductEvent, and UI Snapshot family major `2`. The version constants remain `1.0` until M7 begins the release train; changing those constants and crate versions is a separate, explicit release commit.

## Intentionally unavailable internal surfaces

The following are not stable API migrations; they are deliberate removals of internal parallel paths:

- global provider registry bridges;
- retired agent-loop wrappers;
- direct `IntentRouter` operation admission;
- `CodingAgentSession` convenience pass-throughs that bypass canonical `run`/scheduler dispatch;
- replay handles that bypass snapshot/reconnect recovery;
- legacy internal `ClientConnection`/`SubmittedOperation` projection;
- arbitrary plugin Flow extension declarations;
- raw service access from plugin/tool hosts.

Downstream code that reached these internals must migrate to the stable facade and canonical operation/snapshot/product-event contracts rather than request replacement exports.
