# Migrating `pi-rust` From 0.3.0 To 0.3.1

`0.3.1` hardens the `pi-ai` provider runtime. Most consumers receive stricter
failure behavior without changing call sites, but the intentional public
surface corrections below may require code changes.

## Standalone Image DTOs Are No Longer Stable API

`pi_ai::api::images` has been removed because `0.3.0` exposed request and
response DTOs without a production provider/client execution path. Do not
import those DTOs as an advertised image-generation capability. Conversation
content still supports `ContentBlock::Image` for multimodal model requests.

Standalone image generation can return only after it has a provider-neutral
execution contract with authentication, cancellation, timeout, retry, error,
usage, and cost semantics.

## Codex Accepts SSE Only

`StreamOptions.transport = Some("sse".into())` remains valid for supported
streaming APIs. `"websocket"` and unknown values now return a typed
`UnsupportedOption` provider error before network I/O. The unused Codex
WebSocket URL/frame helpers have been removed.

## Terminal Streams Fail Closed

Provider EOF is no longer interpreted as successful completion. A successful
stream must observe its provider-specific terminal marker and emits `Done` only
for `Stop`, `Length`, or `ToolUse`. Truncation, malformed wire data, provider
failure, timeout, and cancellation emit one `Error`; cancellation uses
`StopReason::Aborted`.

Custom providers consumed through `complete()` must follow the same rule.
`complete()` now rejects error/aborted `Done` values and EOF without a terminal
event.

## Timeout And Retry Scope

`StreamOptions.timeout_ms` is an end-to-end invocation deadline. It includes
payload hooks, provider-owned credential resolution, request attempts, response
hooks, retry delays, and response-body streaming. Retries share the original
deadline and occur only before provider-neutral output is exposed.

Every explicitly supplied option is now either implemented by the selected API
or rejected before its request is sent. Code that relied on silently ignored
Azure, Bedrock, session, cache, tool-choice, or transport fields must remove the
field or select a supporting API.

## Bedrock Profiles Use The AWS Credential Chain

`AWS_PROFILE` and `StreamOptions.bedrock_profile` now execute the AWS standard
credential provider chain. Explicit bearer tokens remain highest priority;
explicit access key/secret/session-token values take priority over profile
resolution. Partial explicit credentials fail closed.

Credential values and custom authentication headers are excluded from
`StreamOptions` serialization and redacted from `Debug` and response-hook
diagnostics.

## Unknown Model Cost Is Explicit

`ModelCost` and provider-neutral `Cost` include `known`. Dynamic or unknown
pricing uses `known = false` with zero numeric components rather than a negative
sentinel. Consumers must check `known` before treating numeric cost as billed
cost.

Stored transcript usage and product usage events preserve this bit. RPC
`get_session_stats` adds `costKnown`; its additive RPC protocol version is
`2.1`. ProductEvent usage adds `cost_known` when false and advances the additive
ProductEvent protocol version to `2.2`.

## Compatibility Metadata Disposition

Use `pi_ai::api::compatibility::compatibility_field_disposition` to determine
whether a serialized compatibility field is enforced by request conversion,
transport, or parsing, or retained as `CompatibilityDisposition::CatalogOnly`.
Catalog-only fields are descriptive and do not alter outgoing requests.
