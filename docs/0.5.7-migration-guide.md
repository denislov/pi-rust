# Migration Guide: 0.5.6 to 0.5.7

0.5.7 corrects the built-in delegated helper capabilities without changing any
wire protocol, persistence schema, or public Rust API.

## Built-In Helpers

The built-in `explore`, `review`, and `check` profiles now expose `read`,
`grep`, `find`, and `ls`. Previously their empty allowlists made them
reasoning-only despite their read-only code-inspection descriptions.

The helpers still cannot write or edit files, run shell commands, delegate more
work, mutate/read persistent session state, or access product UI authority.
Their effective tools are intersected with the parent operation's frozen
capabilities, so deployments that disable a read-only tool continue to withhold
it from children.

Project or user profiles with the same IDs continue to override the built-ins.
Administrators who intentionally used those IDs for reasoning-only profiles
should keep an explicit project/user override with an empty `tools` list.

## Authorization

Workspace-local read-only calls do not require confirmation. Reads outside the
workspace continue to request `ExternalRead` authorization. `AllowOnce` and
`AllowForOperation` approve an already admitted call; they do not add tools or
filesystem capability to a child that lacks them.

## Protocol Inventory

| Family | 0.5.6 | 0.5.7 |
| --- | --- | --- |
| RPC | `2.1` | `2.1` |
| ProductEvent | `2.2` | `2.2` |
| UI snapshot | `2.2` | `2.2` |

No session, outbox, manifest, recovery, profile-schema, or Extension migration
is required.
