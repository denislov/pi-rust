# 0.5.8 Release Evidence

Release date: 2026-07-22. Workspace package and binary version: `0.5.8`.
Implementation baseline: commit `09a8ef8` plus the scoped 0.5.8 candidate diff.

## Root Cause And Correction

The 120ms spinner sleep was owned by `receive_prompt_source()`, a nested future
inside the main interactive `select!`. Every client ProductEvent that won the
outer selection cancelled and rebuilt that future, restarting its sleep. Under
continuous delivery the spinner tick could therefore be starved indefinitely.
Running operation elapsed labels read the clock only during rendering and
depended on that same tick to request a render, so they froze with it.

The correction gives the main interactive loop one independent 120ms Tokio
interval with missed-tick behavior set to `Skip`. While a PromptTask is running,
the interval advances the spinner and requests one coalesced render. Prompt
source events and client deliveries cannot reset the interval, and idle mode
does not schedule animation work.

Compact Context Operations summaries now omit operation IDs and retain status,
kind, elapsed time, and cancellation affordance. The selected detail surface
still exposes full operation, parent, and root identities; projection,
diagnostic, persistence, and machine-readable contracts are unchanged.

## Regression Evidence
