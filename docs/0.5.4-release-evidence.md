# 0.5.4 Release Evidence

Release date: 2026-07-21. Workspace package version: `0.5.4`.

## Delegation Runtime

- `delegate_agent` and `delegate_team` now keep the original parent tool future
  pending through admission, optional confirmation, child execution, and
  terminal delivery. The parent model receives exactly one typed `completed`,
  `failed`, `rejected`, or `cancelled` result instead of a provisional
  `requested` result.
- Automatic and interactive confirmation share the generation-checked tool
  authorization waiter. RPC approval/rejection wakes the original tool call;
  the 0.5.3 delegation approval operations remain only for restored legacy
  pending facts.
- Recursive children retain exact parent/root operation lineage, delegation
  depth/cycle policy, the child capability snapshot, and child-scoped
  authorization. Child task isolation prevents recursive polling stack growth,
  and parent cancellation aborts the child task.
- Terminal folded state is persisted for automatic, interactive, rejected,
  failed, cancelled, and restored-approval paths. Replay and export reconstruct
  the same bounded/redacted delegation status.

## Fullscreen Child Conversations

- ProductEvent envelopes retain operation, parent, and root identity through
  the interactive bridge. Nested Prompt/member events fold into the direct
  delegated child operation rather than the root transcript.
- The main conversation contains one compact delegation status row. Child
  messages, thinking, tools, results, errors, and authorization previews are
  routed only to the bounded child projection.
- Enter on a delegation row replaces the conversation viewport with the child
  page; Escape restores the main page. The child page is read-only apart from
  scoped permission controls and navigation, so its output cannot enter the
  parent composer or transcript.
- Retention is capped at 32 child conversations, 2,048 queued child UI events
  per conversation, and 1,024 rendered transcript items. Fresh-snapshot
  reconnect hydrates child pages from the bounded retained child-event window.

## Runtime Hardening Found During Release

The TypeScript/Wasm vertical-slice gate exposed that an async epoch ticker can
depend on the same current-thread Tokio executor occupied by a Wasmtime Guest.
The epoch clock now runs on the blocking pool, so Guest cancellation and
deadline yields continue independently of that executor. The real Component
gate passed in `204.12s`; most of that cold debug-path time was synchronous
Wasmtime JIT/regalloc during package preparation, before the invocation
deadline begins.
