# `pi-ai` 0.5.0 Baseline Evidence

Measured on 2026-07-20 with `rustc 1.96.0 (ac68faa20 2026-05-25)` and host
`x86_64-unknown-linux-gnu`. The 0.4.2 baseline is an independent `git archive`
of commit `8ef4cd8d7369aac5e7aaa1eff3d3623e71573292`; it does not contain working-tree
changes. Both clean checks used dedicated directories under
`target/perf-baseline/0.5.0-pi-ai/`.

## Commands

```bash
git archive 8ef4cd8d7369aac5e7aaa1eff3d3623e71573292 | tar -x -C "$baseline_dir"
find crates/pi-ai/src -name '*.rs' ... | xargs wc -l
cargo tree -p pi-ai --edges normal --prefix none | wc -l
CARGO_TARGET_DIR=target/perf-baseline/0.5.0-pi-ai/{baseline,current} \
  /usr/bin/time -f '%e %M' cargo check -p pi-ai
du -sb target/perf-baseline/0.5.0-pi-ai/{baseline,current}
```

The baseline archive resolved the dependency graph from its manifest because
`Cargo.lock` was not tracked at that commit. The current tree used the workspace
lockfile. Dependency-node and source comparisons are authoritative; timing and
target-directory bytes are directional environment-local evidence.
