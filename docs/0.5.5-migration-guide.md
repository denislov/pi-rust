# Migration Guide: 0.5.4 to 0.5.5

The 0.5.5 release hardens operation-tree failure behavior without changing the
delegation result shape, ProductEvent envelope, UI snapshot schema, or RPC
command family introduced by 0.5.4.

## Session Write Errors

The stable Rust `CodingSessionError` enum adds `SessionWriteRejected { message }`
with error code `session_write_rejected`. It means a testable/authoritative
store rejection happened before the requested mutation became durable. Callers
may report or retry the original operation according to normal admission policy;
they must not treat the failure as a committed terminal.

`PartialCommit` remains the result when a durable boundary may have been
crossed, such as session facts/outbox succeeding before manifest refresh fails.
Prompt finalization now preserves that first uncertainty instead of issuing a
second terminal write. On reopen, authoritative terminal facts may prove the
operation committed; in that case no `RecoveryPending` residue remains. If the
facts cannot decide the result, the existing durable recovery workflow retains
ownership.

Code matching `CodingSessionError` exhaustively must add the new variant. Code
matching only `error.code()` may handle `session_write_rejected` as a definite
write failure and continue handling `partial_commit`/`recovery_pending` as
uncertain or recovery-owned states.

## Shutdown And Authorization

Dropping an interactive authorization future now removes its registry entry and
publishes/persists cancellation where possible. Requesting runtime Phase A
shutdown also resolves pending authorization waits so shutdown cannot stall on
an unanswered permission prompt. Phase A still does not abort ordinary provider
work: it closes admission and lets admitted work drain. Final
`CodingAgentSession::shutdown()` cancels any root/child operations still open,
drains them, closes the session writer, and then publishes runtime shutdown.

Adapters do not need a new command. Existing RPC `shutdown` clients may observe
a `tool_authorization_cancelled` event for a pending request before the normal
shutdown response.

## Child Conversation Retention

The numeric bounds remain 32 child conversations, 2,048 queued events per child,
and 1,024 rendered transcript items. When the 33rd conversation arrives while a
child page is open, the active page is pinned and the oldest inactive page is
evicted. Escape returns to the main transcript without reintroducing an
untracked conversation.

## Protocol And Schema Inventory

| Protocol family | 0.5.4 | 0.5.5 |
| --- | --- | --- |
| RPC | `2.1` | `2.1` |
| ProductEvent | `2.2` | `2.2` |
| UI snapshot | `2.2` | `2.2` |

The new error code and shutdown cancellation event use existing additive error
and ProductEvent shapes. Session/outbox/manifest event schemas are unchanged;
there is no persistence migration or protocol negotiation change.
