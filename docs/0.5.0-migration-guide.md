# Migrating To 0.5.0

`0.5.0` is an in-progress breaking release focused on reducing the `pi-ai`
runtime. This guide records accepted removals as they land; it is not yet a
release-completion claim.

## Amazon Bedrock Removal

The built-in Amazon Bedrock provider has been removed. The following surfaces
no longer exist:

- built-in API registration for `bedrock-converse-stream`;
- generated `amazon-bedrock` model catalog records;
- AWS standard credential-chain and SigV4 request signing;
- Bedrock-specific fields on `StreamOptions` and `ProviderAuth`;
- Bedrock-specific cache-retention execution behavior.

Applications that require the removed provider should remain on `0.4.2` or
maintain an independent `ApiProvider` implementation and register it into a
scoped `ProviderRegistry`/`AiClient`. `pi-ai` does not retain compatibility
aliases, dormant wire DTOs, or a feature flag for this provider.

## Private Image Generation Experiment

The private, test-only OpenRouter image-generation DTO and mapper have been
deleted. They were never part of `pi_ai::api` and had no production
client/provider registration path, so no supported public replacement is
required. Conversation image input remains supported through the existing
multimodal content-block contract.

## Downstream Test Harnesses

`pi-agent-core` test-support consumers must remove Bedrock fields from
`StreamOptionsPatch`. Retained provider-neutral and Azure/OpenAI option patching
continues unchanged.

## Stable Facade Audit

The facade audit classified every 0.4.2 export group as follows. No further
facade contraction was accepted: repository-local non-use alone is not enough
to remove an embedding contract.

| Facade category | Exported items | Classification/decision |
| --- | --- | --- |
| `api::model` | `Model`, `ModelCost`, `ModelInput`, `ThinkingConfig`, `all_models`, `calculate_cost`, `get_model`, `get_models`, `get_providers`, `lookup_model` | Active cross-crate model contract; retained. |
| `api::conversation` | `AssistantMessage`, `AssistantMessageDiagnostic`, `ContentBlock`, `Context`, `Cost`, `DiagnosticErrorInfo`, `Message`, `StopReason`, `Tool`, `Usage` | Active cross-crate conversation contract; retained. |
| `api::stream` | `EventStream`, `complete`, `AssistantMessageEvent`, `StreamOptions` | Active agent-runtime/provider boundary; retained after removing Bedrock-only fields. |
| `api::stream::json` | `parse_streaming_json`, `repair_json`, `try_parse_streaming_json` | Intended custom-provider embedding/tool-call assembly contract; retained. |
| `api::hooks` | `ProviderPayloadHook`, `ProviderPayloadHookFuture`, `ProviderResponseHook`, `ProviderResponseHookFuture`, `ProviderResponseInfo`, `ProviderStreamHooks` | Active product hook contract; retained. |
| `api::client` | `AiClient` | Active scoped composition contract; retained. |
| `api::auth` | `ProviderAuthDiagnostic`, `env_api_key`, `EnvProviderAuthResolver`, `ProviderAuth`, `ProviderAuthResolver` | Active authentication injection and embedding contract; retained after removing Bedrock-only fields. |
| `api::provider` | `builtin_provider_apis`, `register_builtins_into`, `ApiProvider`, `ProviderRegistry` | Active product registration plus custom-provider embedding contract; retained with the retired built-in removed. |
| `api::error` | `ProviderError`, `ProviderErrorKind` | Intended provider-neutral error classification contract; retained. |
| `api::transport` | `RetryConfig`, `is_retryable_status`, `parse_retry_after_ms` | Intended custom-provider/product policy inspection contract; retained. |
| `api::compatibility` | `AnthropicMessagesCompat`, `CacheControlFormat`, `CompatibilityDisposition`, `ModelCompat`, `OpenAICompletionsCompat`, `OpenAIResponsesCompat`, `OpenRouterRouting`, `ThinkingFormat`, `ThinkingLevelMap`, `ThinkingLevelValue`, `VercelGatewayRouting`, `compatibility_field_disposition` | Required catalog deserialization and compatibility inspection contract; retained. |
| `api::testing` | `FauxCall`, `FauxProvider`, `FauxResponse`, `FauxState`, `FauxToolCall` | Explicit `test-support` contract used by downstream deterministic tests; retained. |

The only accepted 0.5.0 public breaks are the Bedrock-specific fields removed
from `StreamOptions` and `ProviderAuth`, plus the built-in/catalog capability
removal described above. The image experiment was private and therefore does
not appear in the public API diff.
