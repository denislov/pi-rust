# `pi-rust` 0.2 Compatibility Matrix

This matrix is release input for `0.2.0`. It distinguishes live wire
protocols from durable session storage; their version numbers are intentionally
independent.

## Live Protocols

| Surface | Current version | Accepted peer major | Older major behavior | Owner |
| --- | --- | --- | --- | --- |
| RPC command/response | `rpc 2.0` | `2` | RPC hello rejects the peer with `unsupported_protocol_version` and reports requested/supported versions | `pi-coding-agent::protocol` and RPC adapter |
| Product event stream | `product_event 2.0` | `2` | Consumers must reconnect through a matching RPC/snapshot contract; no legacy event envelope is emitted | `pi-coding-agent::events` and protocol adapter |
| UI snapshot/reconnect cursor | `ui_snapshot 2.0` | `2` | Cursor validation rejects a mismatched snapshot major and requires a fresh compatible snapshot | `pi-coding-agent::runtime::snapshot` |

Minor versions may add optional fields. Unknown required features fail closed.
There is no `1.x` compatibility adapter in the `0.2` runtime.

## Durable Session Storage

| Surface | Current writer | Accepted reader | Compatibility policy | Owner |
| --- | --- | --- | --- | --- |
| Session manifest/event log | schema `pi-rust.session`, version `1` | version `1` | The release keeps the existing writer shape. Unsupported schema/version is rejected with recovery guidance; history is never rewritten in place to simulate compatibility. | `pi-coding-agent::session` |

The live protocol major was bumped because the public event, snapshot, and RPC
contracts changed incompatibly. That does not imply a durable writer bump: P6
represented the required durable facts without changing session version `1`.

## Release Gates

- RPC hello tests prove current-major negotiation and old-major rejection.
- Product event inventory tests prove the closed typed event families and exact
  wire identities.
- Snapshot/reconnect tests prove stream, sequence, and snapshot-major checks.
- Session compatibility fixtures prove supported version `1` logs remain
  readable and recovery rejects unsupported durable formats explicitly.
- `scripts/release-api-snapshots.sh` emits machine-readable rustdoc JSON,
  normalized public-item JSON, workspace metadata, compiler identity, and
  checksums for every active library crate. A checked-in freeze manifest guards
  the normalized projection without committing multi-megabyte raw artifacts.
