## Convergence Principles

### Architecture before intermediate compatibility

An intermediate migration step did not need the complete workspace test suite
to pass when a failure was the understood result of an intentional breaking
change. The required discipline was:

1. compile the affected production owner and direct downstream production code;
2. run the focused contract for the active change;
3. classify every other failure as expected convergence debt or a regression;
4. record expected debt with an owner, target phase, and exit condition;
5. restore full-workspace green before the release gate.

Production code was not reshaped into a compatibility layer merely to satisfy
an obsolete test. A test tied to a deliberately removed path could be narrowly
ignored, unregistered, or commented out with a recorded debt identifier. This
was preferable to permanent architectural debt. Production behavior was never
commented out to hide a failure.

All temporary convergence-test and validation debt was closed before `0.2.0`.
The final tree contains no `#[ignore]`.

### One owner and one stable facade

Each active library crate exposes one categorized facade:

```text
pi_ai::api
pi_agent_core::api
pi_tui::api
pi_coding_agent::api
```

Public availability does not grant every downstream crate permission to use an
item. Boundary tests enforce a category and item allowlist for each active
workspace dependency edge. Test-support APIs are non-default and feature
gated.

## Completion Audit

| Contract | Final evidence |
| --- | --- |
| Facades | Each active crate root exposes only its categorized `api` module; positive consumer and negative visibility contracts pass. |
| Dependencies | Cargo preserves `pi-coding-agent -> pi-agent-core -> pi-ai` and `pi-coding-agent -> pi-tui`; category/item allowlist guards pass. |
| Lower-crate structure | Lower implementation modules are private, grouped by the owners in `architecture.md`, and clean under strict default/all-feature Clippy. |
| Product structure | `crates/pi-coding-agent/src/coding_session/` is absent; `runtime`, `operations`, `services`, `session`, `events`, `plugins`, `tools`, `profiles`, `resources`, `app`, `protocol`, and `adapters` have distinct ownership. |
| Operation runtime | `OperationScheduler`, `IntentRouter`, and `OperationControl` each have one owner; dispatch, admission, identity, cancellation, and lineage guards pass. |
| Durability | `SessionEvent` remains the durable source of truth; repository, transaction, replay, old-session, and interrupted-operation recovery contracts pass with writer version `1`. |
| Product events | The closed `CodingAgentProductEventKind` inventory contains 47 normalized events; the compatibility `CodingAgentEvent` production model is absent. |
| Authorization | `OperationCapabilitySnapshot` is the operation-local authority for model, tool, filesystem, shell, plugin, delegation, generation, and revocation decisions. |
| Adapters | Print, JSON, RPC, and interactive adapters consume product operations/events/snapshots rather than Flow, agent, service, or session internals. |
| Tests | Direct integration targets are `3/8/8/11` for `pi-ai`/`pi-agent-core`/`pi-tui`/`pi-coding-agent`; final source-level cases are `188/223/245/1,251`. |
| Release | API freeze, compatibility matrix, migration guide, crate changelogs, strict Clippy, full tests, binary version, offline TUI smoke, and diff checks pass. |

Test counts are an audit measurement, not a quality target. New tests must
protect a distinct high-value behavior at the owning layer. The detailed test
value and exclusion rules are normative in `architecture.md`.

## Closed Convergence Debt

| ID | Resolution |
| --- | --- |
| `CONV-TEST-001` | Core and TUI integration targets that need deterministic harnesses now declare `required-features = ["test-support"]`; normal dependency builds keep testing facades hidden. |
| `CONV-TEST-002` | Product internal tests use one crate-local environment guard owner and explicit deterministic model/configuration inputs. |
| `CONV-TEST-003` | Filesystem-tool E2E fixtures now provide a capability-scoped non-persistent runtime cwd; the temporarily ignored cases were restored and pass. |
| `CONV-VALID-001` | Strict workspace Clippy diagnostics were resolved without broad lint suppression. |

There is no open convergence debt at this release boundary.

## Release Evidence

The final release gate is:

```bash
scripts/release-gates.sh
```

It covers formatting, all-target/all-feature compilation, focused architecture
contracts, public API freeze verification, strict workspace Clippy, full
workspace tests, exact binary version output, offline TUI smoke, and
`git diff --check`.

Checked-in public API manifests live under [`api-snapshots/`](api-snapshots/).
Raw multi-megabyte rustdoc and normalized public-item artifacts remain generated
under `target/release-artifacts/0.2.0/`; the checked-in manifests preserve their
hashes and generator contract without committing those build artifacts.
