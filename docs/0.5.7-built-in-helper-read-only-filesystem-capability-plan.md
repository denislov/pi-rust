# Built-In Helper Read-Only Filesystem Capability Plan For 0.5.7

## Status And Scope

Target workspace version: **`0.5.7`**. Status: **Completed 2026-07-22**.

This scoped successor to 0.5.6 fixes the mismatch between the documented
read-only behavior of the built-in `explore`, `review`, and `check` delegated
profiles and their empty tool allowlists. It does not introduce runtime
capability escalation or relax child-operation isolation.

## Decision

Each built-in helper declares exactly `read`, `grep`, `find`, and `ls`. Its
effective child tools remain:

```text
helper profile allowlist ∩ parent frozen tool capabilities
```

Filesystem capability is released only when that intersection contains a
filesystem tool. `write`, `edit`, `bash`, delegation, session read/write, and UI
capabilities remain absent. Workspace-local reads remain automatically allowed;
external reads use the existing operation-scoped authorization path. An
authorization decision never mints a missing capability.

## Acceptance

- All three built-in helpers expose exactly `read`, `grep`, `find`, and `ls`.
- A permissive parent releases filesystem plus those four tools to each helper,
  but no shell, mutation, delegation, session, or UI authority.
- A restricted parent releases only the intersection; profile declarations do
  not expand parent authority.
- A real delegated `explore` turn reads a workspace fixture and receives the
  result, while an unrelated parent-only tool remains hidden.
- User/project profiles continue to override built-ins through the existing
  registry precedence rules.
- RPC `2.1`, ProductEvent `2.2`, and UI snapshot `2.2` remain unchanged.
- Public Rust APIs remain checksum-identical to 0.5.6.
- Full release and Extension gates pass, and the committed worktree is clean.

## Non-Goals

- Do not grant `bash` to `check`; shell execution is not intrinsically
  read-only and requires a separate policy decision.
- Do not add a capability-request protocol or let authorization widen a frozen
  child snapshot.
- Do not grant write/edit access or persistent session access to helpers.
- Do not change delegation depth, confirmation, retention, or child TUI
  behavior.
