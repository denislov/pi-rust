# 0.5.6 Release Evidence

Release date: 2026-07-22. Workspace package and binary version: `0.5.6`.
Implementation baseline: commit `15dbb9b` plus the scoped 0.5.6 candidate diff.

## Visual And Responsive Contract

The frozen vocabulary, thresholds, semantic roles, and marker ownership are in
[`architecture/fullscreen-visual-contract-0.5.6.md`](architecture/fullscreen-visual-contract-0.5.6.md).
The implementation proves:

- composer assistance, support prompts, Context-owned detail, and modal dialogs
  have explicit placement roles. Slash and file-completion content aligns with
  the composer text origin at 60, 80, 120, and 160 columns; modal geometry and
  focus capture remain independent;
- `ShellLayout` owns Conversation, Context, Tips divider/body, composer,
  assistance, and final status rectangles. Wide Context is a persistent rail,
  medium Context is a bordered drawer, and narrow Context is a composer-free
  page. Exact 63/64/65 and 99/100/101 thresholds plus 0/1/2/8/20-sized
  viewports remain bounded;
- Tips are capability/focus-aware and priority selected. Focused panels,
  list/autocomplete rows, the composer prompt, and transcript selection use
  distinct markers that also work without color;
- dark, light, basic-color, and color-disabled modes resolve the existing theme
  token vocabulary for focused/inactive structure, tabs, selection, runtime
  state, metadata, and context pressure. The final status row uses the shared
  context ratio and never displays `session_label` or a durable session ID;
- System, Assistant, Tool, Error, delegation, and child-page content share one
  reserved transcript gutter. Child headers show concise target/status/ID and
  `Esc back`; child messages, tools, and authorization remain isolated from the
  main transcript;
- Markdown tables bypass paragraph whitespace rewrapping after exact allocation.
  Natural/shrunk/narrow tables retain visible-column boundary alignment across
  ANSI, tabs, links, CJK, combining text, and emoji, with a deterministic narrow
  fallback.

Product-level responsive matrices and tiny/threshold geometry are deterministic
unit/virtual-terminal tests. Child-page and child-authorization appearance is
also tested through real product projection plus virtual-terminal rendering:
the offline production binary has no legitimate child to display without a
provider, and no fake provider-only product path was added merely for tmux.
