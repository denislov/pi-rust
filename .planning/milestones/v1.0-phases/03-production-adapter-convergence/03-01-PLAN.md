<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/process input -> product operation | Untrusted prompt/options become a typed admitted operation. |
| Product events/outcome -> JSON or terminal output | Runtime results cross into externally consumed serialization and error text. |
| Persistent print -> session storage | Prompt completion writes durable semantic facts and manifests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering / Elevation of Privilege | JSON/print operation admission | high | mitigate | Submit only public Prompt operations through `run` and add a production-source guard against workflow bypasses. |
| T-03-02 | Repudiation | JSON event ordering and persistent facts | medium | mitigate | Preserve subscription-before-run, final drains, exact event assertions, and session-log parity tests. |
| T-03-03 | Information Disclosure | adapter error projection | medium | mitigate | Retain existing typed error conversion and serialized fields; add no new diagnostic path or credential output. |
| T-03-04 | Denial of Service | unexpected public outcome | low | accept | Closed-enum mismatch is an internal invariant and is handled with an exhaustive invariant branch rather than retry or new protocol behavior. |
</threat_model>
