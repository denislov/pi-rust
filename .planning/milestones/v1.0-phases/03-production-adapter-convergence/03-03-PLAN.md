<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RPC input -> typed mutation | Profile IDs, delegation reasons, edit requests, plugin command IDs, and arguments cross admission/validation. |
| Plugin command -> capability-aware runtime | Untrusted plugin input must not gain access to private services or registries. |
| Mutation result/events -> RPC response | Durable or runtime effects must agree with the response, idempotency state, and restored owner. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-09 | Tampering / Elevation of Privilege | mutation admission | high | mitigate | Preserve existing validation and submit only public typed operations through canonical admission. |
| T-03-10 | Elevation of Privilege / Information Disclosure | plugin load/command | high | mitigate | Use public plugin operations and curated outcomes; keep services, registries, and raw options private. |
| T-03-11 | Repudiation | owner/event/idempotency ordering | medium | mitigate | Restore owner and drain events on every path before idempotency completion; retain structured tests. |
| T-03-12 | Denial of Service | malformed command or unexpected outcome | medium | mitigate | Preserve typed input errors and exact outcome invariants without retry loops or detached work. |
</threat_model>
