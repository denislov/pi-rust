<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Navigation intent -> persistent owner mutation | Target leaf/session choices cause durable branch summary and fork effects. |
| Pre-navigation subscription -> post-fork owner/projection | Subscriber identity, event sequence, snapshot, and hydration cross a persistent owner transition. |
| Source guard -> architectural enforcement | Textual checks must reject real bypasses without flagging legitimate local UI/lifecycle contracts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-21 | Tampering / Repudiation | summary/fork durable ordering | high | mitigate | Enforce ReuseExisting summary before ForkSession on the same owner and assert durable facts/replay-visible state. |
| T-03-22 | Denial of Service / Repudiation | subscriber and event sequence continuity | high | mitigate | Keep one receiver across operations, final drains, monotonic sequence assertions, and snapshot recovery/hydration. |
| T-03-23 | Tampering | owner replacement/runtime loss | high | mitigate | Return the same mutated owner and verify plugin/profile/runtime state after fork rather than reopening a replacement. |
| T-03-24 | Elevation of Privilege | adapter bypass/private imports | high | mitigate | Complete narrow production guards for canonical calls, public imports, and local deprecation suppression. |
| T-03-25 | Denial of Service | source-guard false positives | medium | mitigate | Sanitize comments/strings, report locations, and explicitly allow legitimate UI/lifecycle methods while deferring parser hardening to Phase 5. |
</threat_model>
