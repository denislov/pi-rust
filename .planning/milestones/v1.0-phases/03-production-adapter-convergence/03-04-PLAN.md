<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI intent -> owned async operation | Synchronous input state transfers the live owner to a background task. |
| ProductEvent/control channels -> UI projection | Concurrent controls and events affect transcript, status, and completion ordering. |
| Plugin operation -> visible extensions | Capability-aware plugin results become menus, dialogs, commands, and notices. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-13 | Denial of Service / Tampering | async owner lifecycle | high | mitigate | Reuse the existing take/task/result/finish lifecycle and always return the same session owner. |
| T-03-14 | Repudiation / Denial of Service | event/control multiplexing | high | mitigate | Preserve select branches, pre-run subscription, final drain, sequence projection, and abort tests. |
| T-03-15 | Elevation of Privilege | plugin actions | high | mitigate | Submit public plugin operations only and retain curated public outcomes/extension projection. |
| T-03-16 | Tampering | direct branch-summary policy | medium | mitigate | Lock AlwaysCreate, no hydration, visible output, and durable facts in a focused test. |
</threat_model>
