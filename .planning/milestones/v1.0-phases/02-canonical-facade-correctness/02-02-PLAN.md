<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public operation -> private operation | Typed caller input crosses into private admission and execution contracts. |
| Private metadata -> dispatcher selection | Incorrect classification can route mutation through the wrong admission/control semantics. |
| Private outcome -> public projection | Internal data must be reduced to the correct stable public family without losing error/result meaning. |
| Test expectations -> implementation under test | Correlated expectations can falsely certify an incorrect metadata or projection row. |

</threat_model>
