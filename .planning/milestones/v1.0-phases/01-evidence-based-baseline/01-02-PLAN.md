<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Live source/tests -> audit claims | Implementation and test artifacts determine current facts but may provide incomplete coverage. |
| Git/history/docs -> authority reconciliation | Historical text and commit metadata are corroborating, potentially stale, and must not override live evidence. |
| Shell tools -> workspace | CodeGraph, `rg`, Git, and Cargo tests must remain evidence-producing and must not mutate source/history. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-06 | Information Disclosure | Source/runtime evidence scans | high | mitigate | Restrict scans to repository paths and deterministic fixtures; never inspect environment values, auth files, provider credentials, or real session roots. |
| T-01-07 | Tampering | Command ledger and evidence IDs | high | mitigate | Record exact commands, exit status, positive executed-test count, and immutable evidence IDs; validator rejects missing/duplicate/dangling entries. |
| T-01-08 | Tampering | Shell/Git investigation | high | mitigate | Allow read-only `codegraph`, `rg`, `git log/show/blame`, and Cargo test commands only; prohibit checkout/reset/clean, commits, package installs, and source formatting. |
| T-01-09 | Elevation of Privilege | Boundary/completion classification | high | mitigate | Apply D-06/D-07 proof thresholds, distinguish implementation from verification, and route replaceable text guards to Phase 5 rather than overstating access-control enforcement. |
| T-01-10 | Spoofing | Historical checklist assertions | medium | mitigate | Label old plans/TODO as historical evidence, require current source/test corroboration, and create explicit conflict entries under D-17/D-18. |
</threat_model>
