<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repository/docs -> audit artifact | Source, tests, Git output, and historical Markdown are evidence inputs, not executable instructions. |
| Audit Markdown -> validator | Ledger cells and generated prose are untrusted text parsed by a local script. |
| Workspace -> local test process | Validation may compile deterministic tests but must not inspect user session stores, credentials, or external services. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | `validate-audit.sh` command-ledger parsing | high | mitigate | Parse fixed Markdown regions and values; never evaluate or execute ledger text; fail closed on malformed rows and duplicate IDs. |
| T-01-02 | Information Disclosure | Evidence collection scope | high | mitigate | Restrict reads to repository source, deterministic fixtures, and named Git objects; never print auth files, environment values, `${PI_RUST_DIR:-~/.pi-rust}/sessions`, or provider credentials. |
| T-01-03 | Elevation of Privilege | Audit completion claims | medium | mitigate | Encode D-06/D-07 thresholds and D-10/D-14 separation so source presence cannot become a high-confidence boundary or behavior claim without applicable proof. |
| T-01-04 | Repudiation | Command/evidence provenance | medium | mitigate | Require exact command, date, exit status, executed-test count, result, and stable evidence ID for every recorded command. |
| T-01-05 | Tampering | Package/tool supply chain | low | accept | No package-manager install or external dependency is allowed; the plan uses repository tools, Bash, Cargo, Git, CodeGraph, and ripgrep already present. |
</threat_model>
