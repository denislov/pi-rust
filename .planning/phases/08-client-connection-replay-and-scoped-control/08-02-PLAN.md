<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Connection handle -> ClientService | Caller-controlled id/generation/draft/control identifiers request state mutation. |
| ClientService -> session projection | Client-local state must not become durable session truth or leak across sessions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-STALE-HANDLE | Elevation of Privilege / Tampering | ClientService mutation methods | high | mitigate | Compare id+generation first on every read/mutation/ack/control-preparation path; takeover tests enumerate old-handle failures. |
| T-08-02-CROSS-SESSION | Information Disclosure | ClientService ownership | high | mitigate | Per-CodingAgentSession field, no globals, constructor-isolation tests. |
| T-08-RESOURCE-BOUND | Denial of Service | draft queues and receipt records | high | mitigate | Private deterministic caps, non-evicting accepted receipts, and typed rejection tests; new unique receipt at 64 rejects before mutation. |
| T-08-02-ROLLBACK | Tampering / Repudiation | submitted status and ack cursor | high | mitigate | Monotonic transition functions reject regression/identity mismatch and retain terminal until matching ack. |
</threat_model>
