<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External embedder -> pi_coding_agent::api | Untrusted caller supplies ids/cursors/text and must receive stable projections only. |
| Public projection -> private runtime | Conversion must not leak authority-bearing services, queues, senders, or private events. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-01-API-SPOOF | Spoofing | CodingAgentClientId/control value types | medium | mitigate | Use typed ids plus later generation/owner validation; document that client id is identity, not user authentication. |
| T-08-01-API-LEAK | Information Disclosure / Elevation of Privilege | api re-export list and public signatures | high | mitigate | Exact facade-set and signature guards reject ProductEvent, raw controls, services, channels, queues, metadata, and Flow types. |
| T-08-01-AMBIGUITY | Repudiation | recovery/control enums | high | mitigate | Exhaustive typed reason/receipt branches with stable fields; tests forbid behavior that depends on display strings. |
| T-08-01-SUPPLY | Tampering | Cargo dependency surface | low | accept | This plan adds no dependency and must leave Cargo.toml/Cargo.lock unchanged; existing locked toolchain is sufficient. |
</threat_model>
