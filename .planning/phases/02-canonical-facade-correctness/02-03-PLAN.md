<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Canonical operation -> persisted session mutation | A typed operation may append durable facts, update the manifest, publish events, and replace owner state. |
| Durable log -> reopened owner state | Replay is authoritative when manifest/publication fails after append. |
| Runtime/plugin/profile/delegation services -> public outcome/events | Internal capability and queue state must be projected without leaking private contracts or changing semantics. |
| Test failure injection -> shared filesystem/global process state | Deterministic faults and environment/provider guards must remain test-only and isolated from concurrent tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-05 | Tampering / Repudiation | navigation/profile/delegation persistence | high | mitigate | Assert outcome, durable facts, typed errors, explicit PartialCommit, and reopen/replay authority through canonical run. |
| T-02-06 | Denial of Service / Tampering | failure injection and global fixture state | high | mitigate | Reuse existing serialized test guards and owner-only fault controls; add no production hook or unsynchronized global mutation. |
| T-02-07 | Information Disclosure / Elevation of Privilege | plugin capability and service projection | high | mitigate | Execute commands through capability-aware PluginService and assert only curated public outcomes/diagnostics. |
| T-02-08 | Repudiation | ProductEvent sequence continuity | medium | mitigate | Assert semantic events and strict sequence monotonicity across owner replacement and applicable mutations. |
</threat_model>
