<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model/tool payload -> protocol events | Untrusted text, JSON-like tool arguments, and errors are serialized for machine clients. |
| product event stream -> RPC queue | Sequenced events enter a bounded client-facing queue with overflow recovery. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-04 | Tampering | `CodingProtocolEventAdapter` | high | mitigate | Match typed fields directly and retain exact serialization/payload assertions; never interpolate payloads into control decisions. |
| T-07-05 | Denial of Service | RPC event queue | high | mitigate | Leave bounded capacity and Overflow-to-fresh-snapshot behavior unchanged and run the existing RPC regression suite. |
| T-07-06 | Information Disclosure | protocol projection | medium | mitigate | Project only established public fields and keep private compatibility/internal types out of RPC values. |
</threat_model>
