<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Spawned interactive task -> main TUI loop | A live mutable session owner and operation error cross an async channel; losing or duplicating ownership corrupts continuity. |
| Canonical durable mutation -> failure projection | Append/manifest failures may leave replay-authoritative state and must retain the exact `PartialCommit` error and owner. |
| Forked owner -> prompt context/root projection | Session identity crosses from runtime persistence into subsequent request routing and visible hydration. |
| ProductEvent -> UiEvent/fallback notice | Runtime events are filtered/projected before becoming user-visible feedback. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-26 | Tampering / Denial of Service | `PromptTask` completion channel | high | mitigate | Make success and failure carry exactly one live owner; restore before error projection; exercise four deterministic failure paths. |
| T-03-27 | Repudiation | durable mutation error handling | high | mitigate | Preserve original canonical errors including `PartialCommit`, operation IDs, and replay authority while returning the owner. |
| T-03-28 | Tampering | forked session target synchronization | high | mitigate | Update prompt context only from a successful `SessionForked` owner and assert old-target retention on failure. |
| T-03-29 | Repudiation / Information Disclosure | delegation visible-event fallback | medium | mitigate | Base fallback on the established UiEvent projection and retain existing notice/error text without duplicate application. |
| T-03-30 | Tampering | structural boundary tests | medium | mitigate | Replace magic counts with per-function diagnostics and retain all canonical-operation/private-import/deprecation guards. |
</threat_model>
