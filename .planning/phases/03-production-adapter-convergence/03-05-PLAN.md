<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Menu/dialog input -> durable mutation | Profile IDs and rejection reasons affect persistent/runtime state. |
| Sync TUI handler -> async owner task | Incorrect ownership can block the event loop, lose the session, or update UI before success. |
| Delegation ProductEvent -> visible decision state | Event loss can leave queue, transcript, and fallback notices inconsistent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-17 | Tampering / Elevation of Privilege | profile/delegation admission | high | mitigate | Preserve typed validation and submit only public profile/delegation operations. |
| T-03-18 | Denial of Service | sync-to-async owner transfer | high | mitigate | Use the existing owner-returning task lifecycle; forbid blocking and detached work through tests/source review. |
| T-03-19 | Repudiation | delegation event/fallback projection | medium | mitigate | Subscribe before mutation, drain through CodingEventBridge, and assert fallback only when the stream yields no UI event. |
| T-03-20 | Tampering | premature local projection update | medium | mitigate | Apply root/prompt profile changes only after the canonical outcome succeeds and verify persistence/reopen. |
</threat_model>
