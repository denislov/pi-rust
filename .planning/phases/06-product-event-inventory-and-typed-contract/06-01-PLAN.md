## Trust Boundaries
| Boundary | Description |
|---|---|
| Internal event -> public contract | Compatibility-era runtime data crosses into a stable embedding and serialization surface. |
| Stable API facade -> downstream caller | Only curated product facts may become externally matchable; internal services and storage facts remain private. |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-06-01 | Tampering / Spoofing | event family and kind identity | high | mitigate | Exhaustive typed conversion and explicit Serde/as_str names replace Debug-derived wire identity. |
| T-06-02 | Information Disclosure | public payload types | high | mitigate | Select owned stable fields per variant; prohibit CodingAgentEvent, services, Flow nodes, and session-log internals in the public contract. |
| T-06-03 | Repudiation | durability and terminal metadata | high | mitigate | Keep sequence, operation ID, terminal status, terminal operation, and durability as independent tested dimensions. |
| T-06-04 | Denial of Service | receiver/order behavior | medium | mitigate | Leave EventService sequencing, retention, channel capacity, and error handling unchanged; projection is pure and one-to-one. |
| T-06-SC | Tampering | dependency supply chain | high | mitigate | Add no dependency; use workspace Serde and Rust test infrastructure only. |
</threat_model>
